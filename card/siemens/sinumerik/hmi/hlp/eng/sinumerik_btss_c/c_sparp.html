block Part program blocks from the current operation
Column index:
4 		Line index:
BlockIndex
Block index, 1 = last, 2 = current, 3 = next block 		Max. line index:
3
To display the currently active part program, the NCK supplies up to 3 blocks of the part program (previous, current and next block) with a variable request.

Line index 1: string from the previous block
Line index 2: string from the current block
Line index 3: string from the next block

In order to obtain consistent information, all 3 field elements must be requested simultaneously in the variable request. For this reason, the maximum string length of each field is limited to 66 characters.
Note: With the EES function, only the current block is supplied. With the help of the BTSS variables workPandProgName, as well as seekOffset and byteOffset, an alternative block display can be implemented on the HMI side.

singleBlock Single block read in quasi-stop state
Column index:
5 		Line index:
BlockIndex
Block index, 1 = last, 2 = current, 3 = next block 		Max. line index:
3
In most cases the variable 'block' is used to read the currently active blocks of the part program. Because this variable is limited to 66 characters per string, it might be necessary (for long blocks) to read longer strings. The variable 'singleBlock' can read complete blocks (up to strings with 198 charecters) . 3 lines can be addressed:

Line index 1: last block
Line index 2: current block
Line index 3: next block

It is not guaranteed for rapid block changes, that the information of 3 successive blocks is always consistent, because each block is read with a single variable request. This method is only safe, if the part program has stopped.
Type
TYPE_STRING 		Unit
No unit 		Default Value
 Minimum Value
 Maximum Value
Access Path HMI: /Channel/ProgramInfo/singleBlock[u<Channel numb>, <Line Index>]
