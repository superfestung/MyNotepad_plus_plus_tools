Block C/SPARPF: Program pointers for block search and stop run

To search for a specific block within a part program, the user can enter search criteria and start a block search. These input values are summarized in the block SPARPF and must be written by the HMI.
1 main program level and 17 subroutine levels can be processed. These levels are the line indices of the individual variables. The search targets (seek pointer and search string) can only be used in a mutually exclusive manner in one level. In case of collisions, a negative acknowledgement is made when the search is triggered.

Depending on the search type, the search string is either a block label, a block number, or any string.

If no path name is specified, then the default search strategy for subroutine calls is used. The main program entered in the 1 program level must be selected for the search; otherwise the search request is acknowledged negatively.

The array indices (line indices) have the following meaning:

1 = main program level for search run
2-18 = subroutine levels for search run

101 = main program level for stop run
102-118 = subroutine levels for stop run

haltBlock Hold block
Column index:
10 		Line index:
1
Program levels (only defined for level 0) 		Max. line index:
1
The following applies to the SPARPI: The interrupt pointer does not mark the block
where the program was cancelled but a previous block (hold block), which enables a better resumption.
The hold block is explicitly set with the part program commands IPTRLOCK and IPTRUNLOCK,
or implicitly manipulated via $MC_AUTO_IPTR_LOCK.
The following applies to the SPARPF: If SPARPI is copied completely,
the value of the hold block which was set by the NCK is retained.
This enables the NCK to recognize the situation, and it responds with the suppressed message alarm 16950.
NOTE: For SPARPI and SPARPF this value is ONLY defined for program level 0.
