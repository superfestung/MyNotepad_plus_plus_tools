Block C/ETP: Types of events

Description of the logging event types.
For this block, access is allowed via several lines and columns.

The line index identifies a specific event.
Standard events: line index < 10000:
OEM events: line index > 10000:
User index: Determined by the 1000s digit of the line index.
Event type: Determined by the last three digits of the line index

Examples for the line index:
00001: Standard event of user 0 with number 1 (IPO)
00006: Standard event of user 0 with number 6 (NC start)
03006: Standard event of user 3 with number 6 (NC start)
06006: Standard event of user 6 with number 6 (NC start)
10001: OEM event of user 0 with number 1
13002: OEM event of user 3 with number 2

Standard event types:

Cyclic events:
1 = IPO and IPO cycle
15 = IPO2
47 = IPO3
48 = IPO4

Acyclic events, related to axis motions:
2 = GEO_AXIS_START and Geo axis starts or changes direction
18 = GEO_AXIS_STARTa see VDI interface NCK->PLC channel-specific
DBB40 Bit6 and Bit7 (Bit6 = traversing command+, Bit7 = traversing command-)
Event occurs when a bit is reset.
3 = GEO_AXIS_STOP and Geo axis stops
19 = GEO_AXIS_STOPa, see VDI interface NCK->PLC channel-specific
DBB40 Bit6 and Bit7 (Bit6 = traversing command-, Bit7 = traversing command+)
Event occurs when both bits are set to 0 and one of them was active before.
4 = MA_AXIS_START, A machine axis of the channel starts or changes direction
see VDI interface NCK->PLC axis-specific
DBB64 Bit6 and Bit7 (Bit6 = traversing command-, Bit7 = traversing command+)
Event occurs when a bit is reset.
5 = MA_AXIS_STOP, A machine axis stops
see VDI interface NCK->PLC axis-specific
DBB64 Bit6 and Bit7 (Bit6 = travel command-, Bit7 = travel command+)
Event occurs when both bits are set to 0 and one of them was active before.

asciiMode Data logging format
Column index:
7 		Line index:
Event (see module header) 		Max. line index:
siehe Bausteinkopf
Data logging format

Value Range:
0: Data recorded in binary format with fixed alignment to 8 bytes
1: Data recorded in ASCII format
2: Data recorded in binary format with variable alignment
3: Data recorded in binary format with variable alignment and optimization of two consecutive data records of the same event. In this case, only the header is logged, not the actual data.
Type
TYPE_UWORD 		Unit
No unit 		Default Value
0 		Minimum Value
0 		Maximum Value
3

headerType Type of header in the data record
Column index:
21 		Line index:
Event (see module header) 		Max. line index:
siehe Bausteinkopf
Type of header in the data record

Value Range:
0: No header
1: Short header with the following structure:
UDword dataStamp; // Data record identified by a consecutive number
UWord event; // Enter type of event that occurred
UWord protCount; // The number of times the event has been logged
2: Long header with the following structure:
UDword dataStamp; // Data record identified by a consecutive number
UWord event; // Enter type of event that occurred
UByte chan; // Channel in which the event occurred
UByte dummy1; // Still free
UDword protCount; // The number of times the event has been logged
UDword dummy2; // Still free
3: Mid-length header, non-aligned with the following structure:
UDword dataStamp; // Data record identified by a consecutive number
UWord event; // Enter type of event that occurred
UByte chan; // Channel in which the event occurred
UByte dummy1; // Still free
UDword protCount; // The number of times the event has been logged
Type
TYPE_UWORD 		Unit
No unit 		Default Value
1 		Minimum Value
0 		Maximum Value
3
Access Path HMI: /Channel/ProtocolEvent/headerType[u<Channel numb>, <Line Index>]
