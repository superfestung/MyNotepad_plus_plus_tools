Configuring and performing an acceptance test

Sequence: Performing acceptance tests

The "Overview" setting area is automatically activated after the acceptance test function is called.

 

1.

Enter the required machine data in the "Overview" setting area.

 

2.

Press the "First test" softkey.

The “Switch-off signal path" setting area becomes active.

If you do not wish to use this test, in this setting area, activate option "This test is not required for this machine". Also in other setting areas you can deactivate the particular test in the same way.

 

3.

Press the "Start test" softkey.

A list of all of the activities required for the test are shown to the right in the detailed display.

 

4.

Parameterize the criteria for the corresponding test type.

 

5.

Press the "Collect data" softkey.

Then follow the instructions on the screen (e.g. activate the forced checking procedure of the switch-off signal paths).

 

6.

In some test cases, you must manually exit data acquisition by pressing the "Finished" softkey.

The alarms that have been acquired are then listed in the detailed display.

In most cases, in field "Collected alarms", each alarm is listed through two channels (NC/drive).

 

7.

Press the "Go to results" softkey.

After the acceptance test, the alarms that occurred are listed.

Depending on the particular test type, a trace with the relevant signals can be evaluated.

 

8.

Define which of the listed alarms should be subsequently documented in the acceptance report.
Remove the checkmark (left) for all alarms that should not be documented in the report.

 

9.

Enter the criteria to initiate and evaluate the test.
An entry is mandatory here, in order to be able to start additional steps.

 

10.

Then evaluate the test result.

Check whether the expected alarms have been received. Depending on the particular test type, a trace is created that you can evaluate.

Make a selection in the bottom right of the detailed view to define whether the test (in this case: the switch-off signal paths) was successful.

The test name is allocated a green checkmark if the test was evaluated to be positive. In this case, you can either perform another test in the active setting area - or as is normally the case, you can initiate the next test via the next setting area.

If the test was evaluated to be negative, the test name is assigned a red cross and you can repeat the test, if necessary.

 

11.

When required, correct the standard test names.

 

12.

To switch to the next setting area, press the "Next test" softkey.

A test is created in the next setting area. A list of the necessary activities for this particular test type is shown in the detailed display.

 

13.

Follow the instructions on the screen (generally these involve steps 3 to 12).

After all tests of the individual setting areas have been performed, at the end, the "Finished" setting area is displayed. In this setting area, you can generate the acceptance report from the acceptance test and save it.

Generating and saving an acceptance report

After completing all of the acceptance tests, in the "Finished" setting area, you can generate a certificate for the acceptance report and save it.

 

1.

Activate the corresponding option in the detailed display if you wish to integrate a certificate into the acceptance report.

 

2.

Press the "Generate certificate" softkey.

The standard file archive is displayed.

 

3.

Select a directory or create a new one.
Then press the "OK" softkey to confirm your selection.

You are prompted to assign a name for the future certificate.

 

4.

Enter the certificate name here.
Then press the "OK" softkey to confirm your entry.

The certificate is then saved in the selected archive folder as RTF file. As a result of the RTF format, the certificate can be subsequently supplemented, e.g. to include a machine diagram.
