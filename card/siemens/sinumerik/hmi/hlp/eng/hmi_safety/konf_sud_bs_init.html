SafeUserData - Initialize blocks

After creating and loading a new project, the user blocks are not initialized. The signature of the user blocks has the value "0". With an initialization, the value "0" can be written to the elements of all initialized user blocks and confirmed. The user block then has a valid signature, and the active values of the elements are available for the user program.

Requirements

  • At least one user block has signature "0H".

  • The "writeEnable" signal of the non-initialized user block is active.

    Initialization is not possible without an active signal.

Call

MENU SELECT > Commissioning > Menu forward key > Safety > SafeUserData > Vertical extension key

The program checks the user blocks.

Vertical softkeys

Name

Description

Initialization

Using this softkey, you start the check on user blocks without signature. The softkey is only activated if at least one of the listed user blocks has no signature.

Call initialization

MENU SELECT > Commissioning > Menu forward key > Safety > SafeUserData > Vertical extension key > Initialize

After checking the user blocks, initialization can be started.

Vertical softkeys

Name

Description

OK

You finally start initialization using this softkey.

The result of the initialization is then displayed.

You confirm the displayed export result using the same softkey.

Cancel

You cancel the initialization using this softkey.

After cancellation, screen form "SafeUserData - Overview" is displayed.

Result

The program attempts to initialize the signatures of all user blocks where up until now the signature was missing. Message "Initialization is completed" is displayed if this operation was successful. In this case, the signature of all user blocks is valid.

On the other hand, if the initialization was not successful, message "Initialization is canceled" is output. Possible causes are:

  • The "writeEnable" signal is not active for all user blocks.
