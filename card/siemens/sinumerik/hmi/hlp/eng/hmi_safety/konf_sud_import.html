SafeUserData - Import

SUD actual values can be imported into the PLC from the selected import file.

Requirements

  • The hash value contained in the file is correct and indicates the integrity of the file.

  • Type and sequence of the elements in the user blocks match the loaded PLC project.

    Slight differences for element names are tolerated.

  • All "writeEnable" signals are active.

Call

MENU SELECT > Commissioning > Menu forward key > Safety > SafeUserData > Import

A dialog opens for selecting the SUD import file.

Information

Name

Description

Import SafeUserData

Here, select the required import file.

Vertical softkeys

Name

Description

OK

Using this softkey, you start the import of SUD actual values from the selected import file. The program checks the import file data and imports all blocks with identical symbol names.

A query appears for inconsistencies between the symbol names of existing data and data to be imported. Define what the response should be to these inconsistencies.

The import result is then displayed.

You confirm the displayed import result using the same softkey.

Cancel

Using this softkey, you cancel the import of the actual values.

After cancellation, screen form "SafeUserData - Overview" is displayed.

Result

The program writes all instances, and in so doing checks the actual values.

Note

Acceptance test required

Always perform an acceptance test after actual values have been imported.

  • If the test was successful, a mask is displayed with a summary of the test. The import can be started with "OK" or canceled with "Cancel". Once the import has been completed, message "Import of SafeUserData is completed" is displayed.

  • On the other hand, if the import was not successful, message "Import of SafeUserData was canceled" is output. Possible causes are:

    • An error occurred when reading the import file.

    • The hash value of the file is not correct.

    • The file version is not a valid.

    • Number/type/name of the user blocks or the sequence of the elements of a block do not match the current PLC project.

    • The "writeEnable" signal is not active for all user blocks.

  • If the check indicated that the import file contains fewer blocks than the active SafeUserData data set, then all identical blocks are imported from the import file. Blocks that are only included in the active SafeUserData data set are not overwritten.

  • If the check indicated that the import file contains more blocks than the active SafeUserData data set, then all identical blocks are imported from the import file. Blocks that are only included in the import file are not overwritten.

  • If the check indicated that the symbol names of the active SafeUserData data sets differ from the symbol names of the import file, then this is displayed in a message (see SafeUserData - Overview - Details). You can define the next steps using softkeys:

    • Cancel softkey

      You can completely cancel the import of the import file.

    • Reject softkey

      You evaluate the differences (OK softkey), which are displayed in a screen form. You decide not to accept the values of the deviating symbol names for the displayed instance. In this case use the Reject softkey. If additional differences for other instances exist, then these are displayed in the subsequent screen forms.

    • Accept softkey

      You evaluate the differences (OK softkey), which are displayed in a screen form. You decide to accept all values of the deviating symbol names of all instances from the import file.

    An overview again shows the decision that you have made. With the OK softkey you can initiate the final import.
