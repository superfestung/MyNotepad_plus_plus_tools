SafeUserData - Check newly activated values

Call

MENU SELECT > Commissioning > Menu forward key > Safety > SafeUserData > Details > Change > Accept

After the modified values have been accepted, you must compare the values in column "Newly activated value" with the values in "Entered value". After the check, start the confirmation sequence using softkey "Confirm/Save".

Warning

Unexpected response due to data not being saved.

A configuration of a data set that is not saved is replaced by the previous configuration the next time the PLC starts. This means that it is possible that an unexpected configuration becomes active, which no longer matches the machine or the machine expansion stage.
As a consequence, this can result in an incorrect response or failure of the safety equipment. Death or severe injury may result if the appropriate precautions are not taken.

  • Ensure that the data save operation was performed.

  • After each configuration change, always restart the PLC and perform an acceptance test.

(SUD38)

Information

Name

Description

Element name

The names of all block elements are listed in this column.

Format

The display format is displayed in this column (B = bool, +/-D = Int/DInt).

Newly activated value

The values active in the PLC are displayed in this column.

Entered value

The newly entered value is displayed in this column.

Vertical softkeys

Name

Description

Confirm/save

You start the confirmation sequence at the PLC using this softkey.

The PLC confirms the values, the block signature is updated and saved.

Cancel

You cancel the change sequence using this softkey. The activated values are reset, and the previously valid values are again active. The confirmation sequence with the PLC is not started.

After cancellation, screen form "SafeUserData - Changing the active values" is displayed.
