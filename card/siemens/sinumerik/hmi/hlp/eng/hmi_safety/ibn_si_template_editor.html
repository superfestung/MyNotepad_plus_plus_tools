Using the template editor

In the "Safety acceptance - template" window you can load the customizing templates for the acceptance test. The customizing templates are created and/or processed for an acceptance test using the SINUMERIK OPERATE template editor.

A customizing template is a user-specific XML file for an acceptance test, in which all of the required test types and instances – together with the corresponding parameters, limit values, alarms and handling instructions – are completely specified.

As a consequence, using a customizing template, an acceptance test can be precisely coordinated for a specific machine and/or machine types.

From the tests specified in a customizing template, during an acceptance test, a deviation can only be made to the extent that it is possible that individual tests are not performed. Additional test types or instances cannot be added.

Call

MENU SELECT > Commissioning > Menu forward key > Safety > Acceptance test > Template editor

Settings

The following inputs can be defined in a customer-specific template:

  • Test types

  • Test instances

  • Test parameters

  • Limit values

  • Problem alarms, i.e. alarms that must occur - or alarms that must not occur

  • Filtered alarms, i.e. alarms that are neither displayed nor logged

  • User-specific tasks to describe a test with instructions for the operator

Customer-specific template (SinuComNC)

An existing customizing template, created using SinuComNC or the CT Editor, can be loaded into the acceptance test and an acceptance test performed. It is not possible to edit the template.
