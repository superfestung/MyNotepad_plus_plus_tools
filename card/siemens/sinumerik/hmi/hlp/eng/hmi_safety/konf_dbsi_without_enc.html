Mechanical configuration/actual value acquisition

Precondition

The "Settings - mechanical configuration/actual value acquisition" window is only available if beforehand an extended safety function without encoder was selected for the actual drive (see Options). The commissioning mode must be activated to change the configuration.

Settings

The "Settings - mechanical configuration/actual value acquisition" window comprises two setting ranges, which you can display by appropriately scrolling:

Setting range "Mechanical configuration"

The following settings are possible:

  • Axis type (p9502)

    Either a linear axis or a rotary axis/spindle can be defined here as axis type.

  • Motor type (p9507.2)

    Sets the function configuration for the safety-related motion monitoring functions. Sets the motor type for the actual value acquisition.

  • Spindle pitch (p9520)

    Sets the ratio between the encoder and load in mm/revolutions for a linear axis with rotary encoder.

  • Actual value tolerance (p9542)

  • Gearbox stage switchover (p9501.26)

    Up to 8 gearbox stages are available with the gearbox stage switchover function. Only a static ratio can be used without gearbox stage switchover. Sets the enable signals for the safety gearbox switchover for the safety motion monitoring functions.

  • Gear stages 1...8

    • Number of load revolutions (p9321)

      Sets the denominator for the gearbox between the encoder and load.

    • Number of encoder revolutions (p3522)

      Sets the numerator for the gearbox between the encoder and load.

  • Direction reversal (p9539)

    Using a control box, the direction of rotation reversal can be defined for each gearbox stage of an encoder.

Only the "Motor pole pair number" (r0313) is displayed.

Setting range "Configuration actual value acquisition"

The following settings are possible:

  • Actual value acquisition delay time (p9586)

    Sets the delay time in ms for evaluating the encoderless actual value acquisition after the pulses have been enabled.

  • Actual value acquisition filter time (p9587)

    Sets the filter time in ms for the actual value smoothing for actual value acquisition without encoder.

  • Fault tolerance (p9585)

    Sets the plausibility monitoring tolerance for the current and the voltage angle.

  • Speed tolerance (p9787)

    Sets the filter time in ms for the actual value smoothing for actual value acquisition without encoder.

  • Actual value acquisition minimum current (p9588)

    Sets the minimum current for encoderless actual value acquisition referred to 1 A as a percentage.

  • Acceleration voltage tolerance (p9589)

    Sets the acceleration limit to filter discontinuities in the speed/velocity as a percentage.

Only the velocity actual value (r9714[0]) is displayed.

The "Pulse enable" and "Actual value acquisition supplies valid actual value" are only marked using LEDs (blue = active; gray = inactive).

Recommended values

As an alternative to the manual input in the displayed input fields, you can go to a dialog by pressing the "Recommended values" softkey. Here, you can accept the recommended values from the encoder data of the axis machine data. The data are then copied from the encoder parameterization into the 1st and 2nd monitoring channels. You can accept these recommended values by pressing the "Yes" softkey. You can exit the dialog with "No" without accepting the values.

Note

It is not possible to use the "Recommended values" softkey for drives without NC axis assignment. Instead of this, the "Copy encoder data" softkey is visible. In this case, standard encoder parameters can be copied to the safety encoder parameters. You can accept these recommended values by pressing the "Yes" softkey. You can exit the dialog with "No" without accepting the values.

Parameter list

When required, each setting in this dialog window can also be made via a parameter list. You can access the parameter list using the ">>" softkey followed by the "Parameter list" softkey.
