SafeUserData - Change active values

You can change the values in screen form "SafeUserData - Change". Softkey "Accept" is grayed out as long as nothing has been changed. The changed values are saved in the internal SINUMERIK Operate memory. To transfer the changed values, softkey "Accept" must be pressed (see "Accepting activated values").

Warning

Incorrect response when configuring an incorrect data set

Configuring an incorrect data set, also on a different machine, can result in the incorrect response or failure of the safety equipment. Death or severe injury may result if the appropriate precautions are not taken.

  • When identifying and configuring SafeUserData, already ensure that the operator panel is connected to the correct NCU/PLC, and the correct data set is selected.

  • To avoid configuring an incorrect SafeUserData data set, just ensure that the data set to be configured is assigned a "writeEnable" signal (this should be initiated in the user program).

  • To identify the correct data set, the name of the data set is displayed. Check these names.

  • To identify the correct NCU/PLC, the serial numbers of the NCU memory cards can be checked in the version data, for example. Check the version data.

  • Carefully ensure that only the data set to be configured in the NCU/PLC to be configured is assigned a "writeEnable" signal. Only one machine is simultaneously configured in the network. As a consequence, configuring a SafeUserData data set can prevent an incorrect NCU/PLC.

(SUD37)

Call

MENU SELECT > Commissioning > Menu forward key > Safety > SafeUserData > Details > Change
