SafeUserData - Export

After defining an export file name, you can select which user blocks should be taken into consideration for the export.

Requirements

  • All "writeEnable" signals are inactive.

  • All SUD user blocks have been commissioned (signature != 0H).

Call

MENU SELECT > Commissioning > Menu forward key > Safety > SafeUserData > Export

The "Export SafeUserData" dialog then opens.

Information

Name

Description

Name

Enter the name of the export file here. The output format is defined as "*.sud".

 

All data set blocks are displayed once a name has been assigned. You can select which blocks should be saved in the export file.

Vertical softkeys

Name

Description

OK

Using this softkey, you start the export of SUD actual values to the defined export file.

The export result is then displayed.

You confirm the displayed export result using the same softkey.

Cancel

Using this softkey, you cancel the export of the actual values.

After cancellation, screen form "SafeUserData - Overview" is displayed.

Result

Message "Export of SafeUserData is completed" is displayed if the export was successful.

Message "Export of SafeUserData is canceled" if the export was not possible. Possible causes are:

  • The "writeEnable" signal is active for at least one block.

  • Not all blocks have been commissioned.

  • Not all blocks have been commissioned and not all "writeEnable" signals are inactive.

    Remedy: Initialize blocks

  • An error occurred when reading the active values of the block elements from the PLC.

  • An error occurred while writing to the export file.
