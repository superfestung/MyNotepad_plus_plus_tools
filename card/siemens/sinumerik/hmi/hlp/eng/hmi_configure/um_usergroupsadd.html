Creating local user groups

Introduction

If you want to create a new local user group, you can use the "Create local user group" dialog.

Requirement

  • You have the role "Security admin manufacturer" or "Security admin end user".

  • The actual domain is not connected with a central user management.

Notice

Local user groups are deleted when connecting to central user management

If a previously locally created user group has corresponding name on the central user management domain, then this local user group assigns users from the central user group their rights via machine roles.

This is the reason that all local user groups that do not have a corresponding name on the central user management domain will be deleted as soon as you connect their active domain to a central user management.

  • If you create additional security administrators besides the standard security administrators of the domain, then we recommend that you directly link this user with machine role "Security Admin".

Procedure

  1. In the "Setup" operating area, call the user management via the "Security > User management" softkeys.

  2. Click the "User groups" softkey. Click on "+ User group" on the overview page to create a new local user group. A pop-up dialog opens.


    Creating a new local user group

  3. Assign any name for the local user group and a description that is as fitting as possible that connects the users of this group.

    Note

    Observe the following notes when selecting a name:

    • Special characters are not permitted when assigning a name for a local user group.

    • In the manufacturer domain, it is not permissible that user groups have the same names as machine roles.
    Note

    Note that the user groups in the manufacturer domain must not have the same name as the machine roles.
    Note

    Note that special characters are not allowed when specifying a name for a local user group.

  4. If you have already created machine roles, select one machine role or several machine roles to which the user group will be assigned.
    The members of this user group are given the highest access level of this or these machine role(s). If no machine roles has been assigned to the user group, the members of the user group do not have any rights (Keyswitch0).

  5. If you have already created users, select users who are members of this local user group.

  6. Click on "OK" to add the new local user group to the list of user groups. If necessary, create additional local user groups by repeating steps 2-6.
