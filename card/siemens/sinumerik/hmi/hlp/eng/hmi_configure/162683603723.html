Use time zone

To ensure that the controller has the correct time information even in countries with different time zones, you can select the appropriate time zone.

The time zone set here is used by the entire system to convert UTC time (Universal Time Coordinated) into local time and to activate and deactivate switchover to daylight saving time.

Note

UTC time as default setting

If check box "Time zone" is not activated, then selection list "NCU/PU time" is displayed write-protected. As default setting, in this case, time zone "Etc/Universal (UTC)" is selected. With this setting, no switchover is made to daylight saving time.

Procedure

 

1.

Select the "Setup" operating area.

 

2.

Press the "HMI" softkey.

 

3.

Press the "Date/Time" softkey.

The "Set Date and Time" window opens.

 

4.

Activate check box "Time zone".

The selection list for the time zone becomes active.

 

5.

For SINUMERIK Operate in the NCU: Select the appropriate time zone from selection list "Time zone".

For SINUMERIK Operate on the IPC/PC: Select the appropriate time zone from selection list "NCU / PU time".

In parallel, the current IPC time zone is displayed write-protected in field "IPC time".

 

6.

Confirm your entry using the "Accept" softkey.

The current time zone is applied by the controller.

Note

Softkey "Accept" is only active if you have made changes.

 

7.

Close the window using softkey "OK".

Note

Changing the time zone can take a few seconds.

Note

If you have defined the time zone for "NCU / PU", then in the background, the same time zone setting is set for the PLC (S7-1500). This ensures that the time zone settings are consistent throughout the system, and for example, the time stamp of error messages of different components are recorded and displayed in synchronism with one another.

Note

Time zones without daylight saving time/standard time

When changing the time zone, take into account the local daylight saving time/standard time. For example, regions like China or Japan have no daylight saving time/standard time, but Europe and the USA do.

Aligning the time zone settings between PLC (S7-1500) and HMI

  • If you have made changes, and confirmed these with "Accept" or "OK", then the set time zone setting is automatically transferred to the PLC for synchronization. You are not specifically informed about this.

  • When you press softkey "Cancel", a check is made as to whether the time zone setting of the PLC matches the time zone setting in file basesys.ini. If an inconsistency is identified when making the check, then you are prompted as to whether you wish to correct this inconsistency. When you confirm the query with "OK", then the time zone setting of basesys.ini is transferred to the PLC.

  • If you have made no changes to the time zone settings, and you wish to exit window "Set date and time" with "Accept" or "OK", then a check is also made as to whether the time zone setting of the PLC matches the time zone setting in file basesys.ini.

Note

When aligning the times and settings between the PLC and file basesys.ini, only the settings are compared; however, not the names that are used. You are not notified of any inconsistencies if only the names of the time zone settings differ.

Examples

Hours are added or subtracted from the UTC time. You can set the following time zones, for example:

Europe

Europe/Amsterdam (UTC+1:00)

Europe/Berlin (UT+1:00)

USA

US/Central (UTC-6:00)

US/Michigan (UTC-5:00)

US/Pacific (UTC+8:00)

China

Asia/Hong Kong (UTC+8:00)

Japan

Tokyo (UTC+9:00)
