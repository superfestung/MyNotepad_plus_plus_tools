Overview, supported variables and limits

To create an editable trace configuration, you must start a new and empty trace session.

Procedure

  1. You start a new and empty trace session by clicking on the "New trace" softkey.

    The "New trace session" dialog opens. You obtain the following selection:

    • New NC/PLC trace: This creates an NC/PLC trace session by default.

    • New drive trace: This creates a drive trace session.

    For an NC/PLC trace session, only NC or PLC variables can be inserted into the configuration. Analogously, in a drive trace session, only drive trace variables can be inserted. The content and behavior of the signal table and the signal selection list changes accordingly.

  2. Select the session type and click on "OK".

    A new and empty trace session is created.
Note

For a trace session created in this application, new signals can be inserted, either by entering them into the signal table or by selecting the signal/parameter in the signal selection window (variable selection).

Supported variables

SINUMERIK Operate trace supports:

  • 64-bit servo variables

  • Non-servo NC variables

  • PLC variables and PLC memory addresses from SINUMERIK 840D sl/828D

  • Symbolic PLC1500 variables and PLC memory addresses of SINUMERIK ONE/MCU

  • OEM-PLC variables, imported from symbol files on the SD card

  • GUD variables (global user data) imported from GUD definition files

  • Drive trace parameters

32-bit servo variables are only traceable if they were loaded from a legacy session file. These signals cannot be inserted into a new trace configuration.

PLC1500 variables in multidimensional array format are not supported.

Note

OEM PLC variables, PLC1500 variable names and GUD variables are only imported when the trace application is opened for the first time. Restart the user interface to refresh the variable data, e.g. after integrating a GUD file.

The following PLC memory address formats are supported:

  • Data block (DB)

  • Bit memory (M)

  • Input/output (I/Q or I/O)

M/I/O addresses are supported on SINUMERIK ONE if at least software version V6.21 is installed on the controller. If not, only data block (DB) addresses are supported.

Type modifiers for PLC addresses are supported (e.g.: DB11.DBW11:INT).

Requirements for trace with symbolic PLC1500 variables

Symbolic PLC1500 variables are compatible with the trace application as of the following software versions:

  • SINUMERIK software for ONE as of V6.23 or MC as of V1.23

  • Engineering process software SINUMERIK STEP 7 Toolbox as of V19

  • SINUMERIK ONE PLC Basic Program plus:

    When using this function, use the SINU_Update2 instruction for cyclical data exchange from the SINUMERIK ONE V13.0 system library.

    More information can be found in the instructions SINUMERIK ONE PLC Basic Program plus Function Manual and the online help in the TIA Portal.

  • SINUMERIK PLC Basic Program for ONE as of V6.23 or MC as of V1.23

Total recording buffer size

To diagnose machine problems when setting up the machine and when support is provided in conjunction with debugging axis motion, NC settings and PLC software, SINUMERIK Operate permits a maximum recording size of 500 MB. Trace recording stops when this limit is reached.

You can extend the recording time in the SINUMERIK Operate trace by:

  • Record the signals with a slower sampling rate

  • Reducing the number of signals with high sampling rate (servo signals)

  • Reducing the number of signals or variables to be recorded in general

To record a large amount of data with high sampling rate (servo signals) (more than 500 MB) or to permit recording times of longer than approximately one day, it is recommended to use SINUMERIK Edge applications such as "Capture".

Maximum number of signals in a trace configuration

A trace configuration can encompass a maximum of 32 signals.

The limit for same signal type can be lower than this, for example see the servo signal limit described below.

Maximum number of servo signals in a configuration

The maximum number of servo signals that can be configured depends on the control type settings.

The setting of MD18373 index 5,6,7 defines the maximum number of servo signals and this can be configured.

Typically the limit is set to 10 signals:

$MN_MM_PROTOC_NUM_SERVO_DATA MD18373[5,6,7]=10

An error is displayed in SINUMERIK Operate if too many servo or PLC signals are configured.

You can increase the maximum number of servo signals from the settings as shown in the following example.

$MN_MM_PROTOC_NUM_SERVO_DATA MD18373[5,6,7]=20

Note

In the example, the upper limit is set to 20; however, it cannot be ensured that exact number can be reached.

As the maximum limit depends on multiple factors and might vary in the trace configuration on your control.

You can configure based on your NC and PLC configuration, for example:

  • For the default setting of MD18373[5,6,7]=10

    10 servo signals

    20 PLC signals (bool)

  • For a user-specific setting of MD18373[5,6,7]=20

    17 servo signals

    10 PLC signals (bool)

Maximum number of drive parameters in a trace configuration

A drive trace configuration can comprise a maximum of 8 drive parameters.

Parameters 1 to 4 and 5 to 8 are recorded by the control system via separate sessions.

Note

The start of recording is only synchronized correctly if you configure a start trigger.

You can add more than 8 signals to the configuration if the same drive parameter is added several times with various bit masks.
