Policies

Introduction

If you want to define or adapt security policies for user management, you can do this in the "Policies" dialog. There you can define or change the password guidelines and login guidelines for all users.

Notice

Data misuse

To prevent misuse of data and to control them as securely as possible using the user management, we recommend applying stringent standards for the use of secure passwords.

  • You can obtain tips for the assignment of secure passwords, for example, from the Federal Office for Information Security ().
Note

Asian characters do not count as upper or lower case letters

When defining password guidelines ensure that Asian characters are not evaluated as upper or lower case letters.

  • As a consequence, do not define any restrictions in these categories if the password can potentially contain Asian characters.

Requirement

  • You have the role "Security admin manufacturer" or "Security admin end user".

Change password policies

  1. In the "Setup" operating area, call the user management via the "Security > User management" softkeys.

  2. Click the "Policies" softkey. You see the current settings for the password policies.


    Policies

  3. Click the "Change" button. You can now change the displayed values.

  4. You can define the following settings:

    • Individual password policies: Take possible settings from the table above.

    • Option, specify that after the password is changed, the last X passwords must not be used. This forces the user to define a new password and not to reuse old passwords. This option can be activated or deactivated.

  5. Confirm your settings by clicking on "OK".

Changing login guidelines

  1. In the "Setup" operating area, call the user management via the "Security > User management" softkeys.

  2. Click the "Policies" softkey. You see the current settings for the login guidelines.

  3. Click the "Change" button. You can now change the displayed values.

  4. You can define the following settings:

    • Option, to restrict the number of invalid login attempts. If you limit the number of invalid login attempts and have not defined an additional period for the reset, then the user is immediately locked after a defined number of unsuccessful attempts.

    • You can also define a lockout time that a user must wait after an invalid login attempt before he can attempt a new login.

    • In addition, after a defined period, you can also automatically reset the number of login attempts. The number of invalid login attempts is only decremented by 1 when the period has elapsed. This assumes that the number of login attempts was not already reached beforehand and the user password has not already become invalid. Using this setting, you can additionally influence option "the maximum number of login attempts".

    • Option, a system use notification is displayed in the login screen form. The system use notification is saved as language file in folder "/user/sinumerik/hmi/lng" for each language.

  5. Confirm your settings by clicking on "OK".

System use notification

You have the option of displaying a system use notification in the upper part of the login dialog of the user management. The system use notification can include additional notes, restrictions of use or declaration of consent for all users that log onto SINUMERIK Operate via the user management.

  • The system use notification is saved as language file in folder "/user/sinumerik/hmi/lng" for each language.

    For example, file name for English: "system_use_notif_eng.ts".

You can see the file structure in the following example:

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE TS>
<TS>
<context>
<name>sysusenotif</name>
<message>
<source>SYS_USE_NOTIF_TEXT</source>
<translation>This is an example for a system use notification with an example text</translation>
</message>
</context>
</TS>
