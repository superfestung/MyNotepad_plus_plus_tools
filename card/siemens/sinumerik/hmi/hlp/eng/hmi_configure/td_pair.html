Pair NCU

Introduction

If you want to pair an NCU with an IPC, you can use the "Pair NCU" dialog.

Pairing is required to establish a relationship of trust between the IPC and NCU(s). This is necessary to use security functions in a distributed system because it enables protection of confidentiality and integrity in communication between the IPC and NCU. This makes misuse of data of the NCU simply by replacing the IPC impossible.

Requirement

  • With active user management: You have the role "Security Admin" or access level 2 (service).
    OR

  • Without active user management: You have at least access level 3 (user).

  • The NCU that you want to pair with the IPC is in status "Pairing possible" The NCU to be paired must be configured correctly beforehand for this purpose.

Procedure

  1. In the "Setup" operating area, call the overview page for pairing with the softkeys "Security > Trusted devices".

  2. Select an NCU with the status "Pairing possible".

  3. Click "Pair NCU".

  4. The dialog with the date comparison of IPC and NCU opens. The date display of IPC and NCU must be synchronized because otherwise the generated certificates may be invalid. Confirm the dialog with "OK" if the data of the components are synchronized.
    Alternatively, cancel the dialog and first synchronize the date settings for both devices before starting the dialog again.
    You can set the date of the NCU in SINUMERIK Operate via "Setup > HMI > Date Time".
    You can set the date of the IPC in the Windows system settings.

  5. The data of the configured NCU opens. Confirm that you trust this NCU by verifying the displayed data (NCU name and serial number) and confirming with "OK".


    Pair NCU

  6. To complete pairing, you must enter the manufacturer password of the NCU to authenticate yourself as an authorized person and thus authorize the pairing on the NCU.

  7. Confirm the password with "OK".

  8. If option "Encrypt SD card" was already activated, then the SD card is subsequently encrypted. Confirm the confirmation prompt for SD card encryption with "OK". The SD card of the NCU is encrypted. The system is restarted.

Result

  • The selected NCU has now established a trusted connection with the IPC and you can use the security functions in the distributed system (IPC and NCU).

  • The selected NCU is in status "Paired".

  • On pairing, a pairing certificate was generated for the NCU. The pairing certificate of the NCU can be viewed on the overview page with "View certificate".

  • On pairing, a pairing certificate was generated for the IPC. The pairing certificate of the IPC can be viewed on the overview page with "View certificate".
