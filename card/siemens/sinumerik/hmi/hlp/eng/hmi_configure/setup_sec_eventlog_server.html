Establishing a connection to the Syslog server

Introduction

The Security Eventlog is used to record and provide security events of the SINUMERIK system.

The saved events relevant to security can be sent to a central SysLog server or a Security Information and Event Management (SIEM) according to SysLog standard RFC 5424.

To send the security-relevant events to a central SysLog server, you must establish the connection to the server in SINUMERIK Operate.

Data are transferred encrypted in order to guarantee data security. Encryption and decryption is performed with the appropriate software - the certificates for TLS (Transport Layer Security).

Note

Window "Eventlog settings"

If a SINUMERIK system is distributed via IPC and NCU, then the IPC and NCU must be paired so that the Eventlog settings can be configured.

However, Security Eventlog settings are still displayed for the IPC even if the IPC is no longer paired with the NCU. In this case, you cannot change the Security Eventlog settings.

If a restored DSF archive contains active "Eventlog settings", then the Eventlog settings remain configurable even if the NCU and IPC are not paired.

Note

The Security Eventlog on the IPC does not record the security-relevant events of the Windows operating system. It is recommended to additionally connect a Security Information and Event Management (SIEM) with the Windows event display.

Note

If a Syslog server or Security Information and Event Management (SIEM) is not configured or is not available, then after 500 events, older security events are overwritten in SINUMERIK Operate. The security events are not persisted.

Note

If the pairing between the NCU and the IPC is lost, then the Security Eventlogs of the NCU are not sent to the Syslog server.

Note

If the device is restarted or run down, all security-relevant events on the device are deleted.

Requirement

  • A central Syslog server is available.

  • For a distributed system, IPC and NCU are paired.

    • With active user management: You have the role "Security Admin" or access level 2 (service).

    OR

    • Without active user management: You have at least access level 3 (user).

Procedure

  1. In the "Setup" operating area, call the Eventlog settings via the "Security > Security Eventlog > Settings" softkeys.

  2. Select the "Activate sending of security events (via syslog protocol) to external server" check box.

    The input fields in the "Server connection" area can now be completed.

  3. Complete the input fields "Host name or IP address" and "Port" of the central Syslog server.

    As default, "TLS via TCP" is used as a protocol type.

  4. Click on "Select certificate" to select the server certificate in the local directory or on a USB storage medium.

    Note

    The certificate to be imported must not be password-protected.

  5. Confirm with "OK".

    The server is started. The server status is displayed under "Server connection".
Note

After you have set up the server, you can modify the connection configuration at any time with the "Change" softkey.
