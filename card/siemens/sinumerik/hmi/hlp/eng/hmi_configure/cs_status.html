Certificate store overview

Introduction

All of the X509 certificates that are used are securely stored in the certificate store.

A tabular overview of all the imported certificates can be seen in the overview page of the certificate store. During the import, you can specify a name and select a use. The certificates are structured based on the following meta data:

  • Name: certificate name that can be freely assigned

  • Category: Certificate category: System or OEM

  • Usage: Selected certificate usage

  • Issuer: certificate issuer

  • Valid until: Expiration date of the certificate

The certificate store is available for IPC (Windows) or NCU/PPU (Linux).

Requirement

  • With active user management: You have role "Security Admin" or access level 3 (user).

OR

  • Without active user management: You have at least access level 3 (user).

Selecting a certificate

You can select a certificate in the table and then perform the following actions:

  • Display certificate: Using this dialog, you can view details about the selected certificate, which are not visible in the table.

  • Delete certificate: You can delete a certificate using dialog "View certificate".
