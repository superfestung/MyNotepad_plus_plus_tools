Connecting central end user domains

Introduction

If you wish to connect your local end user domain with a central user management, then you can parameterize the settings using dialog "Connect end user domain with a central user management".

Connecting the domain with a central user management has the advantage that users can be centrally managed on one UMC server. When a user logs into the central user management system, they receive the rights according to the local assignment of their central user groups to machine roles.

Requirement

  • You require one of the available electronic licenses to set up a central user management (UMC):

    • 6ES7823-1UE10-0YA0 (license certificate for 4000 user accounts)

    • 6ES7823-1UE30-0YA0 (license certificate for 100 user accounts)

  • The UMC server version must be 2.13.1 or higher.

  • User management is activated.

  • The local domain for the manufacturer is set up and the security administrator for the manufacturer is set up.

  • A local domain for the end user is set up and the security administrator for the end user is set up.

  • You have the "Security admin end user" role.

Note

Note that users already created in the local domain remain active and can still be used for logging in.

  • Ensure that these user accounts are kept up-to-date or delete these if they are no longer required, in order to avoid any misuse.
Note

Note that all local user groups that do not have a corresponding name on the central user management domain will be deleted as soon as you connect their active domain to a central user management.

This means that you must carefully ensure the following before you connect to the central domain:

  • That you retrieve the admin rights either directly via an assignment to the corresponding machine role
    OR

  • That the user group, for which you have admin rights, exists in the central domain with the same name.

Otherwise the connection to the central domain will not be able to be successfully established.

Procedure

Connecting end user domains with central user management.

  1. In the "Setup" operating area, call the user management via the "Security > User management" softkeys. The overview page for user management opens in the "Configuration" softkey.

  2. Click on the "Connect" button next to the local end user domain.

  3. Assign a name for the central end user domain.

  4. Enter the UMC server address of the central UMC server.

  5. Confirm your entries with "OK".

  6. Check the UMC server fingerprint.

  7. To confirm the connection, you must authorize yourself using the login data of the central administrator. Enter the user name and password of the central administrator and confirm the login dialog with "Connect".

Result

You have now connected the local end user domain with a central user management. Central users of the UMC server can now be assigned the corresponding machine roles via central user groups and log in to the system with the corresponding rights.

You are still logged in to the system as local admin end user.

Further steps

  • You can use existing central user groups and assign these to machine roles.

  • You can create central user groups and assign these to machine roles.

  • You can create local users and assign these to central user groups.

  • You can disconnect the end user domain from the central user management.
