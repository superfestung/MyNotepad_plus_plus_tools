Configuring start and stop triggers for the drive trace

Use the start and stop trigger settings to be able to automatically trace the sequence for certain events with defined drive conditions.

Requirement

A drive trace has been set up.

Procedure

To open the trigger settings, click on the "Start trigger" or "Stop trigger" softkey.

The dialog for the start trigger or stop trigger settings opens.

You have the following recording options:

Start recording

Manual

Recording is started immediately after pressing the "Start trace" symbol.

Through trigger condition

The trace session is activated using a parameter. You obtain a selection of variables by pressing the "Variable list" softkey. The parameter is selected using the "Select" softkey or from the auto-complete list in the trigger condition grid.

Drive object, condition, bit mask and value:

Define for which drive object and under which condition the trace recording should be started.

The following conditions are listed:

  • Positive edge

  • Negative edge

  • On the bit mask

The "Bit mask" field is only visible when selecting condition "On the bit mask". In this case, the bit mask and the value are displayed in the hexadecimal format. If the bit mask is not equal to zero, the traced value is first filtered using the bit mask, and then compared with the value.

Preprocessing time [s]:

The preprocessing time is an extension before or after the required measurement in seconds. A positive value for the prerun time starts recording before the trigger event. A negative preprocessing time extends the recording after the trigger event.

Stop recording

After the acquired data quantity

Recording is automatically ended if the specified maximum free NC memory capacity in kbytes [kB] is reached. The capacity limit is displayed in the dialog window.

After the specified time

The trace recording is automatically stopped when the specified time has elapsed.

When using the specified time as stop trigger, the maximum possible recording time depends on the number of drive parameters in the configuration and on the selected sampling rate. The current sampling rate is displayed next to the time setting. If the "auto" setting was selected in the sampling rate configuration, this value will be automatically recalculated when you change the recording time.

If you set both the recording time and the sampling rate to fixed values and the selected length of time (plus the start trigger pre-run time, if any) cannot be recorded with the selected sampling rate, then an error will be indicated when you try to start the recording.

Note

If you manually stop the trace recording for a drive, the data upload is interrupted and the measured data is not stored.

To flag this response, the "Stop trace" toolbar symbol changes to the "Cancel trace" symbol in the drive trace session.
