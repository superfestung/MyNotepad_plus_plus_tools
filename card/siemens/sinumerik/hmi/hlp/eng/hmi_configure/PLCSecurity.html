Setting PLC security passwords

Introduction

You can save the following PLC security passwords in the "PLC Security" window:

  • Password to protect confidential PLC configuration data

  • Access level password to protect the PLC against unauthorized access

Requirement

You must have the following access levels/roles to set or change the PLC security passwords:

  • If user management is activated: "Security Admin" role or access level 2 (service)

  • If user management is not activated: Access level 3 (user)

Procedure

Access level password to protect the PLC against unauthorized access

  1. Click on the "Enter" button in the "PLC access level password" area.

  2. Enter the password for PLC access protection in the password input field.

  3. Confirm your entry with "OK".

    To display the correct status and use the password that has been entered, the PLC must be restarted after the change.

    Dialog "Restore connection" is displayed.

  4. In dialog "Restore connection" confirm the PLC restart.

Password to protect confidential PLC configuration data

  1. In SINUMERIK Operate, select the "Setup" operating area.

  2. Press the "Security" softkey.

  3. Press the "PLC security" softkey.

    If the passwords are still not set, "Not set" is in the password input field.

  4. In the "Password for PLC configuration data" area, click on the "Enter" button.

  5. Enter the password for the PLC configuration data in the password input field.

  6. Confirm your entry with "OK".

  7. If a password has already been saved in the internal password manager, but not activated in the PLC, then this can be set using the "Activate" button without having to re-enter the password.

    The PLC is stopped.

Note

To display the correct status and use the password that has been entered, the PLC must be restarted after the change. After entering the password, the "Restore connection" dialog opens, in which you confirm the restart of the PLC.

Note

You can delete or change the access level password and the password for PLC configuration data.

Error messages when a password is assigned

The following table lists error messages that can occur when using the access level password:

Display

Error message

Comment

 

Warning

The password for the PLC configuration data is not saved in SINUMERIK Operate, and is therefore not in the backup archive.

This warning is displayed if the password for the PLC configuration data was set in the control system via the TIA Portal. This means that the control system can be used normally. However, if hardware is replaced, the password is not part of the archive, and must be set again.

 

Warning

Not sufficient to generate PLC archives

This warning is displayed if the access level is less than "Full access including fail-safe (no protection)". This access level is required for projects with Safety Integrated in order to be able to create a PLC archive.

 

Error message

Not sufficient to function correctly

This error message is displayed if the access level is less than "HMI access". "HMI access" is required so that SINUMERIK Operate can read and write values in the PLC.

Error message

Password for PLC configuration data was not able to be set in the PLC.

Check the access level and try again.

This error message is displayed if the password cannot be set for the PLC configuration data as a result of an excessively low access level. The access level "full access" is required as a minimum to set the password.

However, an access level password is only required if the project was correctly loaded.

The error could also occur, for example, if you wish to change the password for the PLC configuration data before a project, protected by the password for PLC configuration data, is loaded to the control system.

Restart of the PLC required

After setting or changing the security-relevant passwords, the PLC must be restarted.

If you reject the restart suggested by the system, the warning "Restart necessary" will be displayed. Rejecting the restart suggested by the system has the following effects:

  • When setting or changing the access level password:

    PLC communication available, no connection to PLC possible

  • When setting or changing the password for PLC configuration data:

    PLC communication available, no connection to the PLC possible, PLC in the STOP state
