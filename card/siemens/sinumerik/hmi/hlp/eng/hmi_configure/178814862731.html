SSH server settings

Introduction

To improve the security level of the SSH server on the NCU/PU, in area "SSH server on the NCU" you can make the following settings:

  • Activate/deactivate legacy access for Operate for SSH connections

    Via SSH, SINUMERIK also provides the option of accessing the NCU/PU file system. SINUMERIK Operate on the IPC also uses SSH to access the NCU/PU. For basic commissioning/maintenance, the NCU/PU allows SINUMERIK Operate to be accessed based on a fixed standard key.

    The SSH key is automatically individualized by pairing an NCU/PU with the IPC. This step is not necessary for NCU/PU-based systems.

    However, the SSH standard key can still be used. As a consequence, it is recommended to manually deactivate the standard key to prevent access based on the standard key.

    Caution

    Data misuse by compromising the SSH standard key

    The legacy access for SSH connections via a standard key represents a security risk. If this standard key is compromised (e.g. by an external IPC), then an attacker can access system data.

    • The standard key is not used if you operate the NCU as standalone system.

    • If you operate a distributed system comprising NCU/PU and IPC, then pair the NCU/PU with the IPC. SSH authorization between NCU/PU and IPC is individualized as a result of the pairing and the key is not required.

    • Deactivate the legacy access for SSH connections via the standard key in SINUMERIK Operate.
    Note

    By deactivating the SSH standard key, you exclude all non-paired devices from an SSH connection!

    However, you have the following options if you still wish to establish an SSH connection to non-paired external devices:

    • SSH connection to a non-paired IPC: using software "WinSCP" ™, "PuTTY" ™ or via SINUMERIK Access MyMachine/ P2P

      • With user management activated: User name and password

      • Without activated user management: Access level and password

    • SSH connection to SINUMERIK Edge, SINUMERIK commissioning tool or HMI base application on an external device

      • Import a private SSH key or activate the legacy access in SINUMERIK Operate.

  • You can activate the "Auto-logoff function" of the SSH server after a time that can be individually configured. In this way, you ensure that the SSH connection is not inadvertently maintained for an excessively long time therefore providing an unnecessary attack surface.

The two options can be activated or deactivated independently of each other.

The security host manages the SSH settings and these are saved in the security archive.

Requirement

  • With active user management: You have the role "Security Admin" or access level 2 (service).

    OR

  • Without active user management: You have at least access level 3 (user).

  • Activating/deactivating legacy access for Operate:

    • If you operate a distributed system comprising NCU/PU and IPC: The IPC was already paired with a connected NCU/PU.

  • Auto-logoff function of the SSH server:

    • This function only refers to NCU-only systems of the active NCU.

Deactivating legacy access for Operate

As default setting, the legacy access for SINUMERIK Operate is activated via the standard SSH key.

  1. In the "Setup" operating area, call SSH server settings via the "Security > HMI security" softkeys. The overview page for all HMI security settings opens.

  2. Click on "Change".

  3. Remove the checkmark in check box "Allow legacy access for Operate". This means that you deactivate the standard SSH key for SINUMERIK Operate.

  4. Confirm the settings with "OK".

  5. Restart the system to activate changes to the SSH server settings. If the system is not restarted then the changes only become effective the next time that the system is restarted.

If you wish to reactivate the legacy access, then activate the corresponding checkmark in the check box.

Automatically logging off the SSH server

As default setting, the "Auto-Logoff function" of the SSH server is deactivated if the system was inactive for a longer period of time.

  1. In the "Setup" operating area, call SSH server settings via the "Security > HMI security" softkeys. The overview page for all HMI security settings opens.

  2. Click on "Change".

  3. To activate the function, set a checkmark in check box "Auto-Logoff server connection for inactivity".

  4. Define a period of time after how many minutes of inactivity the automatic logout should take place. The default value is 60 minutes.

  5. Confirm the settings with "OK".

  6. Restart the system to activate changes to the SSH server settings. If the system is not restarted then the changes only become effective the next time that the system is restarted.

If you wish to deactivate this function, then remove the corresponding checkmark in the check box.
