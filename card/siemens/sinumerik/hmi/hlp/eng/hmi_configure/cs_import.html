Importing a certificate

Introduction

Use dialog "Import certificate" if you wish to save a new certificate in the certificate store.

Requirement

  • With active user management: You have role "Security Admin" or access level 3 (user).

OR

  • Without active user management: You have at least access level 3 (user).

Procedure

  1. In the "Setup" operating area, call the certificate store via the "Security > Certificate store" softkeys.

  2. Click on "Import certificate" to import a supported certificate from a local directory or from a USB data storage medium or from the network drive. If it involves an IPC, then you can also import a supported certificate from the hard disk. See below for a list of the supported certificate types. Select the certificate and click on "OK".

  3. If the certificate to be imported is password-protected, then the appropriate dialog is now displayed to enter a password. Enter the password for the certificate and confirm with "OK". The following import dialog is immediately displayed if the certificate is not password-protected.

  4. You can assign the subsequent meta data for the certificate in the following import dialog:

    • Name: Certificate name that can be freely selected.

      Note

      Permitted characters

      Note that a maximum of 64 characters (“A-Z”, ”a-z”, “0-9” , “-”, “_”) are permitted for the certificate name.

    • Usage: Select a usage area for the certificate: Security Eventlog server certificate, Security Eventlog client certificate, Email server certificate. Which certificates can be used for this purpose is determined based on the purpose of use.

  5. Confirm your selection with "OK".

  6. If the certificate and the private key are saved in 2 separate files (see below), you are now prompted to load the key file and enter the associated password.

Supported certificate types

pkcs12

  • pkcs12 certificate file including the private key (only if the file is encrypted)

    Note

    pkcs12 is always supplied with an encrypted bundle comprising certificate and private key.

pem/crt/cer(ASCII)

  • pem certificate including the private key (with encryption of the private key)

  • pem certificate including the private key (without encryption of the private key)

  • pem certificate file (without encryption)

  • pem file with private key (with and without encryption)

cer(binary)

  • cer certificate file (without encryption)

  • cer file with private key (without encryption)

der

  • der certificate file (without encryption)

  • der file with private key (without encryption)

Note

Encryption password

Only ASCII characters for the certificate password/password for the private key are supported.
Note

  • Only X509 V3 certificates are supported.

  • The cer and der format do not support bundled certificate and no private key in a single file.

  • Certificate string with a maximum hierarchy of 4 is supported.

  • Just importing a private key is not supported. This is only supported in the import workflow if the use of a key is required.

Result

The certificate is now shown in the list in the certificate store with the data that have been entered.
