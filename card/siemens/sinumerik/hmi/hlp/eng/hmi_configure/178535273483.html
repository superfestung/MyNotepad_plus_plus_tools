Encrypt/decrypt SD card

Introduction

You can encrypt parts of the NCU SD card to protect data saved on the card (OEM and user data) from unauthorized access and manipulation.

If you have activated SD card encryption, then the data can only be read with the device which was used for encryption. This ensures that sensitive data that you have saved does not fall into the wrong hands if the SD card is lost or stolen.

The SINUMERIK SD card encryption involves a partial encryption, where the most important folders of the SD card are encrypted.

The following folders are encrypted:

  • /oem folder

  • /user folder

Requirement

Note

This function only refers to NCU-only systems of the active NCU.
Note

The Security archives / Security restore password was already set.

The Security archives / Security restore password is a requirement for encrypting the SD card. The function cannot be used if this password is not defined.

  • If you wish to access the data of the encrypted SD card from an NCU/PU other than the NCU/PU that was used to encrypt the data, then you must authenticate yourself in the boot menu by entering the Security archives / security restore password.
Note

Setting not part of the security archive

Note that the SD card encryption setting is not saved in the security archive, and can only be manually activated/deactivated.

  • With active user management: You have the role "Security Admin" or access level 2 (service).

    OR

  • Without active user management: You have at least access level 3 (user).

  • You have sufficient free memory space on the SD card. The free memory should be 1.2 more than the memory occupied by both the /oem and /user folders. The free memory space is required during the activation/deactivation process, after which it can be reused.

Encrypting an SD card

  1. In the "Setup" operating area, call SD card encryption via the "Security > HMI security" softkeys. The overview page for all HMI security settings opens.

  2. Click in area "SD card encryption" on "Activate". A confirmation prompt opens to restart the NCU. The restart takes a somewhat longer time.

  3. Confirm the confirmation prompt with "OK".

The NCU restarts and the SD card is encrypted.

Decrypting an SD card

  1. In the "Setup" operating area, call SD card encryption via the "Security > HMI security" softkeys. The overview page for all HMI security settings opens.

  2. Click in area "SD card encryption" on "Deactivate". A confirmation prompt opens to restart the NCU. The restart takes a somewhat longer time.

  3. Confirm the confirmation prompt with "OK".

The NCU restarts and the SD card is decrypted.
