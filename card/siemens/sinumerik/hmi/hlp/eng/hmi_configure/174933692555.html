Possible errors when configuring the Syslog server

Error cases and possible solutions

The following table contains possible errors that can occur when connecting to the SysLog server and output using error messages in SINUMERIK Operate:

Error case

Possible solution

The server is not available.

The external SysLog server is switched off or there are problems with the network connection to the external server. Check the server status or the network cables.

The certificate is invalid.

Problem with the validity data of the server certificates. Use a certificate that has not yet expired.

The certificate chain is invalid.

The validation of the server certificate fails when creating a connection due to an invalid certificate chain.
The user can check the server certificate chain to resolve this problem.

A signature error occurred.

The server certificate has invalid signatures. Validate the signatures of the certificate to resolve this problem.

The provided server certificate is not suitable for the intended purpose.

The provided server certificate is not suitable for TLS communication. Validate the key usage or the fields for extended key usage in the certificate.

A TLS connection error occurred.

Other TLS connection error. Restarting the SINUMERIK device (IPC or NCU/PU) or the external SysLog server can resolve this problem.
