Displaying the Security Eventlog in SINUMERIK Operate

Introduction

The security-relevant events are displayed in SINUMERIK Operate under "Eventlog". Here, you can search for descriptors that you have logged in the Eventlog.

The connection status is displayed under "Server connection". If there is no connection to the SysLog server, it is displayed under "Server connection" that the pending Eventlog events must be sent to the server.

Requirement

  • With active user management: You have the role "Security Admin" or access level 2 (service).

OR

  • Without active user management: You have at least access level 3 (user).

Procedure

  1. Select the "Setup" operating area.

  2. Press the softkeys "Security > Security Eventlog > Eventlog".

    The security-relevant events are displayed.

    Note

    The display is not automatically refreshed if new security-relevant events occur. Press the "Refresh" button to display the latest security-relevant events.

    New security-relevant events appear at the bottom of the Eventlog.

  3. Press the "Search" button if you want to search for specific keywords in the Eventlog.

  4. Enter the required term, e.g., "USB", in the search form and confirm with "OK".
    The cursor is positioned on the first entry that corresponds to the search term.

  5. If the text that is found is not the specific text that you are looking for, press the "Continue search" button.

  6. Press the "Cancel" button to cancel the search.

  7. Press the "Refresh" button to update the list of displayed hits.

  8. To delete all entries in the Security Eventlog, press the "Delete all" button.

    For a distributed system, the security-relevant events of the NCU and the IPC are deleted if the NCU and the IPC are paired. If the NCU is not paired with the IPC, and a delete operation was initiated on the IPC, then only the security-relevant events of the IPC are deleted.
    All entries that were sent to the server are kept.
Note

To be able to display and operate the "Delete all" button, you must have the following access levels/roles:

  • If user management is activated: "Security Admin" role

  • If user management is not activated: Access level 3 (user)
