System utilization

You have the option of displaying the system resources currently used for NC areas (utilization display).

The following table shows the resources that are displayed for the NC areas:

Component

Description

Total NC load

Shows the load from the cyclic tasks (SERVO position controller and IPO interpolator) and the non-cyclic tasks (PREP preprocessing + external communication).
In addition to the cyclic tasks, the non-cyclic tasks load the processor depending on the demand. Functions such as Top Surface and collision avoidance are executed in the non-cyclic task. The non-cyclic task operates with the processor performance that is available after the cyclic task has been performed.

Reference value current

The current utilization should be below 100% on average

Reference value maximum

A short-term maximum of 100% represents an optimal utilization of the processor and not a critical utilization. For this, consider the load by the cyclic tasks (NC load by position controller and interpolator).

NC load from position controller and interpolator

Shows the load caused by the cyclic tasks (SERVO position controller and IPO interpolator). The cyclic load must be below a certain guide value so that the non-cyclic tasks, such as PREP preprocessing or external communication, are processed.

MD10185 $MN_NCK_PCOS_TIME_RATIO = 90 (default setting)

Reference value current

The current utilization rate should not exceed 50% on average.

Reference value maximum

The maximum utilization should be less than 75% for typical operator actions.

Position controller

Position-control cycle time display:

MD10061 $MN_POSTCTRL_CYCLE_TIME

Interpolator

 Synchronized actions

Interpolation cycle display:

MD10071 $MN_IPO_CYCLE_TIME

To display the time required for synchronized actions, activate MD11510 $MN_IPO_MAX_LOAD. Enter the proportion of the interpolator cycle in %.

Preprocessing

The factor for the interpolator cycle can be set in MD10070 $MN_IPO_SYSCLOCK_TIME_RATIO.

Fill level of interpolator buffer

The channel-specific filling level is displayed in MD28060 $MC_MM_NUM_IPO_BUFFER_SIZE, and indicates whether block preparation for block processing can commence.

Jerky machining in continuous-path mode, if for instance a number of short traversing blocks have been programmed in succession, is a typical indicator of the interpolator buffer idling.

Total system utilization in milliseconds [ms]: MD10050 $MN_SYSCLOCK_CYCLE_TIME (system base cycle)

Note

The utilization evaluation of the synchronized actions is only displayed when the value in MD11510 $MN_IPO_MAX_LOAD ≠ 0.

Procedure

 

1.

Select the "Diagnostics" operating area.

 

2.

Press the menu forward key.

Further horizontal softkeys are displayed.

 

3.

Press the "System utilization" softkey.

The "System Utilization" window opens.

You can track the dynamic utilization display.

 

4.

Press the "Stop" softkey to stop the display update.

 

5.

Press the "Start" softkey to refresh the values.
