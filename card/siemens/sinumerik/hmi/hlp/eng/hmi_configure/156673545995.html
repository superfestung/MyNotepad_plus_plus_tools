Setting the alarm display for 1:N

If more than one NCU can be accessed, then the alarm and message service automatically connects with all accessible NCU systems (server mode). In SINUMERIK Operate, a prefix is set before alarm and message texts for the NCU involved, e.g. NCU_ONE_2: <Message text>.

To suppress the prefix for the alarm and message text display, deactivate server mode in your configuration file "slaesvcconf.xml" in area <ServerMode>.

Without server mode, the alarm and message service only connects to the currently used NCU. Alarms and messages of other systems are not displayed.

You can copy area <ServerMode> from the original file. File "slaesvcconf.xml" is located in the following directory:
/siemens/sinumerik/hmi/base

Setting

Activate/deactivate the server mode in tag <Enabled type="bool" value="TRUE"/>.

Entry

Meaning

TRUE

Alarm and message texts of all NCU systems are displayed with a prefix. (default)

FALSE

Alarm and message texts of other NCU systems are not displayed.

Example

<?xml version="1.0" encoding="UTF-8" standalone="yes"?>
<CONFIGURATION>
 <ServerMode> 
  <Enabled type="bool" value="FALSE"/>
 </ServerMode> 
</CONFIGURATION>
