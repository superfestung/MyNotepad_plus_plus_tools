Physical unit

When a new variable is added to the trace configuration, the application automatically determines the corresponding physical unit. If the displayed unit is preceded with a hashmark"#" symbol, then this means the unit may be incorrect, for example because the axis parameter is not valid.

If a trace session file is loaded in compatibility mode, the physical units are determined based on the current machine configuration. The displayed units may be incorrect or empty (unknown). Invalid units are flagged using the "#" symbol.
