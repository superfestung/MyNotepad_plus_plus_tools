Block T/TMV: Magazine data: Directory

The TMV data module can be used for the following purposes:

1. To display all magazines. The most important magazine information is collected in the TMV module. Existing magazines are sorted in continuous ascending order of magazine number. This means that variables that are defined in this module as 1-dimensional arrays contain all the magazine information without any gaps. The row index with which a specific array can be addressed does not refer to the magazine number, it is just a serial number. Inserting or deleting magazines changes the contents of a row dynamically.

2. Targeted access to magazine data in the TM, TP and TPM modules. Which magazines are actually defined should be determined from the TMV module before accessing an element in the stated modules.

3. Display of all adapters. The most important adapter information is collected in the TMV module. Existing adapters are sorted in continuous ascending adapter number order. This means that variables defined in this module as 1-dimensional arrays contain all the adapter information without any gaps. The row index with which a specific array is addressed has no connection to the adapter number, but is just a serial number. Inserting or deleting adapters changes the contents of a row dynamically.

4. Targeted access to adapter data in the ADN modules. The adapters actually defined should be determined from the MTV module before accessing an element in the stated module.
