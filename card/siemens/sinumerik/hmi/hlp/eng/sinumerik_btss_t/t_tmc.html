magCBCmd Type of current PI service
Column index:
7 		Line index:
 Max. line index:
The variable has the code "1" for the empty location search with the PI services:
These are the PI services:
o _N_TMFDPL
o _N_TMFPBP
o _N_FDPLMT
With the PI services for moving a tool between locations of internal magazines, this variable contains the code for the type of PI service.
If the movement comes from a real magazine or leads to a real magazine, the type of PI service is entered in "magCmd" in the T/TM module.
"magCmdState" can be used to determine whether the commands have been completed.
Access to the status and results of the PI services can be controlled by semaphores.

Value Range:
Value range:
o 1 = _N_TMFPPL, _N_TMFDPL and _N_FDPLMT (empty location search)
o 2 = _N_TMMVTL (move tool)
o 3 = _N_TMPOSM (position tool)
o 4 = _N_POSMT (position multitool)

magCBCmdState Status of current PI service
Column index:
8 		Line index:
 Max. line index:
The processing state of the PI service can be queried with this variable, which is stored in "magCMCmd".
Only the status is determined for PI service _N_POSMT_. No parameters (magCMCmdPar1...) are made available.

Value Range:
Values:
o =0: Initial state (after the control has run up).
o =1: The PI service has been accepted but not yet output to the PLC.
o =2: The command has been output to the PLC, and the NC waits for (end) acknowledgment from the PLC.
o =3: The PI service was terminated successfully. The tool has reached its destination location.
o =4: The PI service was terminated with an error. This also occurs if the parameterization of the PI service is incorrect or the command with the acknowledgment status=3 (without alarm) was cancelled.

magConfMagSearchStrat Type of empty location search in the TO area $TC_MAMP2
Column index:
13 		Line index:
 Max. line index:
Type of empty location search. The variable has a bit-by-bit meaning.
With this variable, the empty location search can be set centrally for the entire TO unit.
Note: When creating a magazine, the settings of $TC_MAMP2 are used first. With $TC_MOP10, these settings can be adjusted magazine-specifically.
In bits 8-15, it corresponds to the variable "magSearch".

Value Range:
A set bit has the following meaning:
Bit8: Search forwards. Search in ascending order from location no. 1.
Bit9: Search forwards. Search in ascending order from the current location at the change position.
Bit10: Search backwards. Search backwards from the last location no.
Bit11: Search backwards. Search backwards from the current location at the change position.
Bit12: Symmetrical search. The search starts at the current location no. at the change position (1st location left, 1st location right, 2nd location left, 2nd location right and so on).
Bit13: 1:1 replacement (only with tool change of significance): If the old and new tools have the same location type and size, the magazine location of the "new" tool to be loaded is transferred to the "old" tool to be unloaded and vice versa. The 1:1 replacement is checked first. If the 1:1 replacement is not possible, the other settings become effective for the search strategy.
Bit14=0: Search first in the individual magazines. If no possible location is found, search for a free location for the tool in the next magazine.
Bit14=1: Search in all magazines for the best location for the tool corresponding to the hierarchy.
Bit15=0: (Conventional type of hierarchy): With this type, the location type of the sought tool is sought in the table of system variables $TC_MPTH. If the location type is found, this hierarchy is accepted, and evaluated from this level to the end.
Bit15=1: (Alternative type of hierarchy): Location type hierarchies can be defined for the selected location types 1, ..., $MN_MM_MAX_NUM_OF_HIERARCHIES. The hierarchy for location type 1 is defined by $TC_MPTH[0,n], that for location type 2 by $TC_MPTH[1,n], and so on. (n: index within a hierarchy). With this setting, one location type can be defined in different hierarchies.
Bit16: The hierarchy consideration is canceled for the mini hierarchy that consists merely of the location type itself and location type 0. In this case, the empty location search does not distinguish between the suitable location type ( $TC_TP7 == $TC_MPP2) and the general location type "0" of the magazine location.
Type
TYPE_UDWORD 		Unit
No unit 		Default Value
 Minimum Value
 Maximum Value
Access Path HMI: /Tool/MagazineConfiguration/magConfMagSearchStrat[u<Tool Area>]
 
magConfToolSearchStrat Type of tool search in the TO area $TC_MAMP2
Column index:
12 		Line index:
 Max. line index:
Type of tool search. The variable has a bit-by-bit meaning.
With this variable, the tool search can be set centrally for the entire TO unit.
Note: When creating a magazine, the settings of $TC_MAMP2 are used first. With $TC_MOP10, these settings can be adjusted magazine-specifically.
Bits 0-7, correspond to the variable "magSearch".

Value Range:
A set bit has the following meaning:
Bit0=0: (Default strategy) Take the first available tool found in the tool group. Search first in the magazine from which the last change was made.
Bit0=1: Select the "active" tool in the magazine of the previously changed tool, otherwise find the replacement tool with the lowest duplo number. If no tool is found in this magazine, the search is continued in the other linked magazines.
Bit1: Find the next replacement tool that is closest to the current magazine position.
Bit2: Select the "active" tool, otherwise the replacement tool with the lowest number contained in $TC_TP10.
Bit3: Find the tool in the group with the lowest actual value of the monitored variable.
Bit4: Find the tool in the group with the highest actual value of the monitored variable.
Bit5: Reserved
Bit6: Search first in the currently considered magazine (effective only in conjunction with bit 7=1).
Bit7=0: Start the tool search in the magazine from which the last changed tool came.
Bit7=1: Always start the search in the 1st magazine in the distance table.
Note:
Bit7=1 + bit0=1 or bit2=1, if no "active tool" is found in the magazine, then - if present - the active tool is selected from one of the other magazines linked to the tool holder
Type
TYPE_UDWORD 		Unit
No unit 		Default Value
 Minimum Value
 Maximum Value
Access Path HMI: /Tool/MagazineConfiguration/magConfToolSearchStrat[u<Tool Area>]

Column index:
4 		Line index:

Type of tool search (bit 0...7) and
type of empty location search (bit 8...15)
With this variable, the tool search and empty location search can be set centrally for the entire TO unit.
Note: When creating a magazine, the settings of $TC_MAMP2 are used first. With $TC_MOP10, these settings can be adjusted magazine-specifically.

Value Range:
Tool search:
o Bit0=0: (Default strategy) Take the first available tool found in the tool group. Search first in the magazine from which the last change was made.
o Bit0=1: Select the "active" tool in the magazine of the previously changed tool, otherwise find the replacement tool with the lowest duplo number. If no tool is found in this magazine, the search is continued in the other linked magazines.
o Bit1: Find the next replacement tool that is closest to the current magazine position.
o Bit2: Select the "active" tool, otherwise the replacement tool with the lowest number contained in $TC_TP10.
o Bit3: Find the tool in the group with the lowest actual value of the monitored variable.
o Bit4: Find the tool in the group with the highest actual value of the monitored variable.
o Bit5: Reserved
o Bit6: Search first in the currently considered magazine (effective only in conjunction with bit 7=1).
o Bit7=0: Start the tool search in the magazine from which the last changed tool came.
o Bit7=1: Always start the search in the 1st magazine in the distance table.
Note:
Bit7=1 + bit0=1 or bit2=1, if no "active tool" is found in the magazine, the active tool - if present - is selected from one of the other magazines linked to the tool holder.

Bit9: Search forwards. Search in ascending order from the current location at the change position.
Bit10: Search backwards. Search backwards from the last location no.
Bit11: Search backwards. Search backwards from the current location at the change position.

Bit13: 1:1 replacement (only significant with a tool change): If the old and new tools have the same location type and size, the magazine location of the "new" tool to be loaded is transferred to the "old" tool to be unloaded and vice versa. The 1:1 replacement is checked first. If the 1:1 replacement is not possible, the other settings become effective for the search strategy.
Bit14=0: Search first in the individual magazines. If no possible location is found, search for a free location for the tool in the next magazine.
Bit14=1: Search in all magazines for the best location corresponding to the hierarchy for the tool.
Bit15=0: (Conventional type of hierarchy): With this type, the location type of the sought tool is sought in the table of system variables $TC_MPTH. If the location type is found, this hierarchy is accepted, and evaluated from this level to the end.
Bit15=1: (Alternative type of hierarchy): Location type hierarchies can be defined for the selected location types 1, ..., $MN_MM_MAX_NUM_OF_HIERARCHIES. The hierarchy for location type 1 is defined by $TC_MPTH[0,n], that for location type 2 by $TC_MPTH[1,n], and so on. (n: index within a hierarchy). With this setting, a single location type can be defined in different hierarchies.

modeWearGroup Strategy wear group $TC_MAMP3
Column index:
11 		Line index:
 Max. line index:
Handling of the tools in a wear group (bit 0...7)
Search strategies for wear groups (bit 8...15)

Value Range:
Effects on the tool status
o Bit 0=0: When a wear group is activated, the status of the tools remains unchanged
o Bit 0=1: When a wear group is activated, the status of the tools changes. A tool is activated in each tool group.
o Bit 1=0: When a wear group is disabled, the status of the tools remains unchanged
o Bit 1=1: When a wear group is disabled, the tools are set to "disabled".
o Bit 2...7 Reserved
Search strategy for the next wear group
o Bit 8=0: Find the next possible wear group
o Bit 8=1: Find the wear group with the next higher, activatable group number
o Bit 9...11 Reserved Search strategy within the tool group for the tool to be set to active (language command SETTA or PI_SETTST)
o Bit 12=0: Lowest possible duplo number
o Bit 12=1: Lowest possible magazine location number
o Bit 13=1: Lowest number contained in $TC_TP10 (replacement order)
Type
TYPE_UWORD 		Unit
No unit 		Default Value
 Minimum Value
 Maximum Value
Access Path HMI: /Tool/MagazineConfiguration/modeWearGroup[u<Tool Area>]
