Block T/MTUD: Multitool data, user-defined data

This block contains the user-defined data of the multitool data.

OEM-defined parameter for the multitools for all permissible data types, except the data type "String".
The OEM data are made available with the MD settings MD bits MD18080 $MN_MM_TOOL_MANAGEMENT_MASK, bit0=1, bit2=1, bit10=1 and MD18192 $MN_MM_NUM_CC_MULTITOOL_PARAM>0.
The data types of the individual parameters are specified by MD18193 $MN_MM_TYPE_CC_MULTITOOL_PARAM, and can be determined from the file _N_COMPLETE_TOA_ACX.
The transfer in the OPI is always as TYPE_DOUBLE.
If there is a read or write access to an OEM-defined multitool parameter of the data type "String", then an error string is returned.
