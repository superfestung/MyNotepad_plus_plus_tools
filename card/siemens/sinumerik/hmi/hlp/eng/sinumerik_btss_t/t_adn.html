Block T/ADN: Adapter data

Adapter data are used to define the dimensions of an adapter and the direction of any loaded tools of each magazine location.
The transformation is applied when cutting edge data are processed in the OPI modules TOT, TOST and TOET if the tool is loaded in a magazine location to which adapter data are assigned.
Adapter data exist independently of magazine location data. The magazine location data contain a reference (see TP module, placeData) to the adapter data.
Adapters can be inserted between a magazine location or a multitool location and a tool.
The adapters change the length and direction of the tool offsets concerned.
The transformation is applied when cutting edge data are processed in the OPI modules TOT, TOST and TOET if the tool is loaded in a magazine location or a multitool location with an adapter to which the adapter data are assigned.

The adapter data describe such an adapter, and exist independently of the use of the adapter in a magazine or multitool location.
The adapter used is determined by the magazine location or multitool location (see TP module, placeData or MTP module, placeData.

The variable is assigned to adapter type 2 (angle head adapter).
An angular offset can be defined for each of the three rotary axes of an angle head adapter.
The angles must be read from the scale on the angle head adapter for the respective rotary axis.
The offset which balances the 0° position of the angle head scale with the initial setting must be added to this angle.
The initial setting corresponds to a chain with the following properties: 1st rotary axis is parallel to the Z-axis (C-axis), 2nd rotary axis points in the direction of the positive X-axis, 3rd rotary axis C1 points at C=0 and A=0 in the direction of the positive Z-axis
