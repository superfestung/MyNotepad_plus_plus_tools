Column index:
11 		Line index:
Magazine number
Magazine number 		Max. line index:
numMagsMax
The processing state of the PI service can be queried with this variable and is stored in "magCmd".

Value Range:
Values:
o =0: Initial state (after the control has run up).
o =1: The PI service has been accepted but not yet output to the PLC.
o =2: The command has been output to the PLC, and the NC waits for (final) acknowledgment from the PLC.
o =3: The PI service was terminated successfully. The tool has reached its destination location.
o =4: The PI service was terminated with an error. This also occurs if the parameterization of the PI service is incorrect or the command with the acknowledgment status=3 (without alarm) was cancelled.

magPlaceSearchStrat Empty location search strategy in the magazine $TC_MAP10 Bits 8-15
Column index:
16 		Line index:
Magazine number
Magazine number 		Max. line index:
numMagsMax
The variable returns the current empty location search strategy of the magazine. It corresponds to system variable $TC_MAP10 bit 8-15.


Value Range:
Value range:
o Bit 8: Forward search as from location number 1 in ascending order.
o Bit 9: Forward search as from the current location at the change position in ascending order.
o Bit 10: Backward search, backwards from the last location number.
o Bit 11: Backward search, backwards from the current location of the change position.
o Bit 12: Symmetrical search beginning at the current location number at the change position (1st location left, 1st location right, 2nd location left 2nd location right, etc.).
o Bit 13: 1:1 replacement (only significant with tool change): If the location types and sizes of the old and new tools are the same, the magazine location of the "new" tool being loaded is transferred to the "old" tool being removed, and vice versa. The 1:1 replacement is preferably checked. If a 1:1 replacement is not possible, the other settings are used for the search strategy.
o Bit 14=0: Search first in the individual magazines. If no possible location is found, search in the next magazine for an available location for the tool.
o Bit 14=1: Search in all magazines for the best location corresponding to the hierarchy for the tool.
o Bit15=0: (Conventional type of hierarchy): With this type, the location type of the searching tool is sought in the table of system variables $TC_MPTH. If the location type is found, this hierarchy is taken and evaluated from this level to the end.
Bit 15=1: (Alternative type of hierarchy): Location type hierarchies can be defined for the recorded location types 1, ..., $MN_MM_MAX_NUM_OF_HIERARCHIES. The hierarchy for location type 1 is defined by $TC_MPTH[0,n], for location type 2 by $TC_MPTH[1,n], and so on. (n: Index in a hierarchy). With this setting, a location type can be defined in various hierarchies.
Type
TYPE_UWORD 		Unit
No unit 		Default Value
 Minimum Value
 Maximum Value
Access Path HMI: /Tool/MagazineDescription/magPlaceSearchStrat[u<Tool Area>, <Line Index>]
 
magPlaceUserDataNumLimit Access to user magazine location data
Column index:
18 		Line index:
Magazine number
Magazine number 		Max. line index:
numMagsMax
The line number for access to the user magazine location data in the TUP module is calculated as follows:
numMagLocParams_u * magNrPlaces. (MD18092 $MN_MM_NUM_CC_MAGLOC_PARAM * $TC_MAP6[magNo] * $TC_MAP7[magNo]).
However, the maximum possible line number is 32767, this means that not all user location data can be addressed. There are (bit-coded) status codes to indicate this state.

Value Range:
Value range:
o Bit0=1: The product of the current values from the number of magazine locations ($TC_MAP6[magNo] * $TC_MAP7[magNo]) and the number of user magazine location parameters (MD18092 $MN_MM_NUM_CC_MAGLOC_PARAM) exceeds the maximum line number. These values may not yet be effective.
o Bit1=1: The product of the effective values from the number of magazine locations ($TC_MAP6[magNo] * $TC_MAP7[magNo]) and the number of user magazine location parameters (MD18092 $MN_MM_NUM_CC_MAGLOC_PARAM) exceeds the maximum line number. Therefore not all user data from all magazine locations of this magazine can be read on the OPI.
After setting the value of $TC_MAP7[magNo], the current and effective values from the number of magazine locations ($TC_MAP6[magNo] * $TC_MAP7[magNo]) are equal.

No unit 		Default Value

 Maximum Value

magToolSearchStrat Tool search strategy in the magazine $TC_MAP10 Bits 0-7
Column index:
15 		Line index:
Magazine number
Magazine number 		Max. line index:
numMagsMax
Tool search strategy within the magazine. It corresponds to the system variable $TC_MAP10 bit 0-7.
If bit7=1 + bit0=1 or bit2=1: If no "active tool" is found in the magazine, then - if available - the active tool is selected from another magazine connected to the tool holder.
Note: When creating a magazine, the settings of $TC_MAMP2 are used first. With $TC_MOP10, these settings can be adapted to the specific magazine.

Value Range:
Search strategy:
o Bit0=0 (Default): First available tool from the tool group. Begins the search in the magazine where the last change took place.
o Bit0=1: Active tool from the magazine of the last change. Otherwise tool with the lowest duplo number. If there is no match in this magazine, the search is continued in the other linked magazines.
o Bit1=1: Next replacement tool nearest to the current magazine position.
o Bit2=1: Active tool with the lowest number contained in $TC_TP10.
o Bit3=1: Search the tool in the group with the lowest actual value of the monitored variable.
o Bit4=1: Search the tool in the group with the highest actual value of the monitored variable.
o Bit5: Reserved
o Bit6=1: Search in the current magazine (only if bit7=1).
o Bit7=0: Begins the search in the magazine where the last tool change took place.
Type
TYPE_UWORD 		Unit
No unit 		Default Value
 Minimum Value
 Maximum Value
Access Path HMI: /Tool/MagazineDescription/magToolSearchStrat[u<Tool Area>, <Line Index>]
 
magWearCompoundNo Active wear group $TC_MAP9
Column index:
14 		Line index:
Magazine number
Magazine number 		Max. line index:
numMagsMax
The variable sets the stated wear group in the magazine [n] to active. The tool locations of the magazine must have previously been assigned to a wear group via variable $TC_MPP5.
Example: $TC_MAP9[1] = 1st wear group 1 in magazine "1" is activated.

Value Range:
o >0: Number of the wear group with which the tool search begins.
o 0: No wear group active
o <0: Number of the wear group with which the tool search begins. However, if this wear group is disabled, the system changes to the next possible wear group for the next tool search.
