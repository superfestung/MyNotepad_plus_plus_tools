Block T/TAPD: Application-specific magazine location data

The TAPD data block is organized as a 2-dimensional variable array. This block contains application-specific data of a T area. Each element can be addressed via a column and row index:

The column index is the magazine number, i.e. one column contains the application-specific magazine location data for all the locations of one magazine. The assignments between magazines and magazine numbers are listed in the appropriate Magazine Directory (TMV) block in the relevant T area. A request is negatively acknowledged if a non-existent magazine number is entered as the column index.

The number of rows is determined by the number of parameters per magazine location and the number of magazine locations:

Max. number of rows = numMagLocParams_tap * magNrPlaces

Application-specific magazine location data are reserved for SIEMENS applications.

The variable returns the Siemens application magazine location data.
Magazine location parameters for Siemens applications for all permissible data types, except the data type "String".
The magazine location parameters for Siemens applications are made available by the MD settings MD bits MD18080 $MN_MM_TOOL_MANAGEMENT_MASK, bit0=1, bit2=1 and MD18202 $MN_MM_NUM_CCS_MAGLOC_PARAM>0.
The data types of the individual parameters are specified by MD18203 $MN_MM_TYPE_CCS_MAGLOC_PARAM, and can be determined from the file _N_COMPLETE_TMA_ACX.
The transfer in the OPI is always as TYPE_DOUBLE.
If there is a read or write access to a magazine location parameter of data type "String" for Siemens applications, an error string is returned.
