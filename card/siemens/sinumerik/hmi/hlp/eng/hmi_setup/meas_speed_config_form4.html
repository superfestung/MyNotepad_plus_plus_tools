Measurement parameters: Disturbance frequency response (disturbance after the current setpoint filter)

The bars in the upper line indicate the traversing path, the actual position and the software limit switches.

The absolute values for start position and end position are displayed in the first two rows of the table.

Status indicates whether the axis is referenced, and whether sufficient space is available to the software limit switch.

If the measurement parameter results in an excessively long traversing distance, so that before the end of the measurement, a software limit switch would be reached, then this is displayed in color and in the "Status" row:

  • "Caution" + yellow: Measurement is possible, but there is only a limited distance remaining.

  • "No reference point + yellow: Axis is not referenced, the user must check the traversing distance.

  • "Fault" + red: Measurement is not possible, as the software limit switch would be passed.

  • "OK" + green: The axis is referenced, and a measurement is possible as the traversing distance lies within the software limit switch.

Note

If the measurement is not possible as a result of the necessary traversing distance, then the following changes are recommended:

  • Change the direction, for example from +,- to -,+

  • If both directions are already selected, for example fewer averaging operations, then repeat the measurement several times.

  • Reduce the offset or reduce the number of repeats.

  • Waveform: Square wave (step)

  • Duration: The duration of the measurement is determined by the number of measurements, averaging operations, offset velocity, settling time and ramp-up time.

  • Distance: Distance traveled during the measurement.

  • Number: Number of measurements which is obtained from the direction and the number of repeats.

  • Direction: Specifies in which direction the measurement should be made, e.g. +,- : first positive, then negative.

  • Repeats: Number of measurements which were defined by the direction.

  • Averages: Number of averaging operations in the time domain which should be executed in one direction during the measurement. The higher the number of averaging operations specified, the smoother the result. However, the measurement duration is extended, and there is possibly insufficient space available. Alternatively, you can reduce the offset velocity.

  • Settling periods: Number of noise periods until the measurement starts.

  • Amplitude and offset velocity can either be entered as an absolute number or as a percentage. The amplitude can be specified to be either positive or negative.

    • Amplitude: Noise amplitude of the PRBS as torque in [Nm].

    • Amplitude %: Noise amplitude of the PRBS as torque in % of p2003

    • Offset: Constant velocity during the measurement in [mm/min].

    • Offset %: Constant velocity in % of p2000 * spindle pitch / gear ratio

  • Settling time: Wait time in [s] after reaching the offset velocity.

  • Bandwidth: The bandwidth of the measurement can be set, and is limited by the speed controller cycle:

    Default: Maximum bandwidth (for a cycle time of 0.125 ms and maximum bandwidth of 4000 Hz)

  • Ramp-up time: Time interval in [ms] in order to reach the offset velocity.

Further actions:

  • Softkey "Recommended" to perform the measurement with the recommended measured values: the entry fields are then deactivated.

  • Softkey "Previous" is only active if a measurement has already been performed. After this, the previous measured values can be restored.

  • Softkey "User-defined" to enter new measured values, with which the measurement should then be performed.

  • Softkeys "Machine data" and "Parameters" switch over from the parameter view to the machine data view – and vice versa.

  • Softkey "Measure" to start the measurement in the "Measurement" dialog (with graphic display).

  • Softkey "Exit" to exit the parameter setting; a measurement is not started.
