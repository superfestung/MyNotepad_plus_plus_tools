Cogging torque compensation - learning run

General conditions for the learning run:

  • Rotary synchronous motors are traversed with a motor speed of 1 rpm and a motor-side traversing distance of 2.2 revolutions.

  • Torque motors are automatically detected based on the drive parameters and traversed with a motor-side speed of 0.5 rpm.

  • No learning cycle is offered for axis couplings and gantry groupings.

Configuring and starting a learning run

Procedure:

  1. A part program is created from the configuration data for the learning run:

    The traversing distance and the velocity for the part program are calculated in such a way that an optimum result is achieved with the learning run.

  2. Open the Part program using softkey "Start >".

  3. Edit the part program using softkey "Change".

  4. Confirm with "OK", and NC start starts the execution of the part program.

  5. Press the "Cancel" softkey to cancel the learning run at the SINUMERIK Operate. An NC reset is required in order to stop the processing of the part program in the NC.

  6. The determined values are written to the compensation table at the end of the learning run and saved in the drive. The compensation values are activated in all drive data sets. Select softkey "Activate in DDS" to only activate certain DDS.

  7. The compensation table values are displayed under softkey "Diagnostics >".

Notice

Criteria for canceling the learning run

The learning run is canceled in the following cases:

  • The traversing distance is too short.

  • The motor-side speed is too high.

  • Reset is initiated.

  • An alarm is output.

If a cancel message is output at SINUMERIK Operate after the learning run has started, the cause can be in the drive. Switch to the "Diagnostics" operating area for more detailed information: Softkey "Alarm list" or softkey "Drive system".
