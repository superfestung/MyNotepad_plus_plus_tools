Measurement parameter axis: Measure speed control loop for checking

The bars in the upper line indicate the actual position and the software limit switches.

The absolute values for start position and end position are displayed in the first two rows of the table.

Status indicates whether the axis is referenced, and whether there is sufficient space to the software limit switch.

If the measurement parameter results in an excessively long traversing distance, so that before the end of the measurement, a software limit switch would be reached, then this is displayed in color and in the "Status" row:

  • "Caution" + yellow: Measurement is possible, but there is only a limited distance remaining.

  • "No reference point + yellow: Axis is not referenced, the user must check the traversing distance.

  • "Fault" + red: Measurement is not possible, as the software limit switch would be passed.

  • "OK" + green: The axis is referenced, and a measurement is possible as the traversing distance lies within the software limit switch.

Note

If the measurement is not possible as a result of the necessary traversing distance, then the following changes are recommended:

  • Change the direction, for example from +,- to -,+

  • If both directions are already selected, for example fewer averaging operations, then repeat the measurement several times.

  • Reduce the offset or reduce the number of repeats.

  • Waveform: Square wave or PRBS

  • Duration: The duration of the measurement as a result of the number of measurements, averaging operations, offset velocity, settling time and ramp-up time.

  • Distance: Distance traveled during the measurement.

  • Number: Number of measurements which is obtained from the direction and the number of repeats.

  • Direction: Specifies in which direction the measurement should be made, e.g. +,- : first positive, then negative.

  • Repeats: Number of measurements which were defined by the direction.

  • Averages: Number of averaging operations in the time domain, which should be executed during the measurement (= one direction). The higher the number of averaging operations specified, the smoother the result. However, this has the disadvantage that the measurement duration is increased. It is also possible that there is insufficient space; in this case, alternatively the offset velocity can also be reduced.

  • Settling periods: Number of noise periods until the measurement starts.

  • Amplitude: Noise amplitude of the PRBS as torque in [Nm]

    The amplitude can be specified to be either positive or negative.

  • Amplitude %: Noise amplitude of the PRBS as torque in % of p2003

  • Offset: Constant velocity during the measurement

  • Offset %: Constant velocity in % of p2000 * spindle pitch / gear ratio

    Amplitude and offset velocity can either be entered as an absolute number or as a percentage.

  • Settling time: Wait time after reaching the offset velocity in [s].

  • Bandwidth: Bandwidth of the measurement in [Hz]; here, the bandwidth is limited by the speed controller cycle.

    Default: For a cycle time of 0.125 ms, the maximum bandwidth is 4000 Hz.

  • Ramp-up time: Interval in [ms] in order to reach the offset velocity.

⇒ Further actions:

  • "Recommended" softkey to accept the values recommended by the automatic servo tuning.

  • "Previous" softkey to accept the values from the previous measurement: This is only possible if a measurement has already been performed and measured values exist.

  • "User-defined" softkey to change machine data that is active during measurement.

  • "Machine data" softkey to display machine data that is changed during the measurement.

  • "<< Back" softkey to return to the Measurement parameters dialog.

Note

Setting the filter

The speed setpoint filters are of no interest when measuring the closed speed control loop. As the Kv factor (MD32200) is reduced, then the position controller should also be stable without speed setpoint filter.

If the position controller still oscillates in spite of the reduction of MD32200, then for instance, you can activate the first speed setpoint filter p1414 = 1H. The horizontal softkey "Active filter" and the vertical softkey "Filter group" → "Speed setpoint filter" can be used to set the filter parameters.
