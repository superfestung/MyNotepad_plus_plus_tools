Topology display: Target topology

When selecting the "Target topology" softkey, configuration data is read in and the configured target topology of the complete drive system is graphically displayed. A DRIVE-CLiQ port is always simultaneously selected for the component in focus. The Drive-CLiQ connection at a selected port is graphically highlighted.

Precondition:

It is assumed that the target topology has no error. As a consequence, the errors are only displayed in the actual topology.

 

If a component has an error, then in addition to the symbol in the graphic, error information is also provided at the bottom right in the detailed view.

Note

Functional scope of the "Topology display"

The functionality under "Drive system" → "Topology display" is a display function with the objective of identifying topology errors.

Contrary to this, in the functionality under "Drive device" → "Topology " (tabular form) the following functions are also integrated:

  • Activating/deactivating drive objects

  • Deleting drive objects

  • Adding and deleting components

⇒ Additional actions: Actions:

  • Softkeys "Drive device +", "Drive device -" can only be used if the system has more than one drive device.

  • Softkey "Actual topology" to switch over the view to the connected components.

  • Softkey "Topology comparison" to switch over the view.

  • Softkey "Zoom >", to adapt the view using zoom +/- or full display.

  • ">>" softkey to switch over the vertical softkey bar:

    • Softkey "Properties" to display additional ordering data, e.g. for spare selected components.

    • Softkey "Identify via LED" to localize the selected component in the control cabinet.
