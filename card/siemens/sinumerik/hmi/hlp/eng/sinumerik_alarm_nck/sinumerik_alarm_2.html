The fourth parameter specifies the type of memory whose limit was exceeded.

- "iS" stands for internal static or buffered user memory. This memory type is defined by the current memory configuration and is not settable. A few NCK functions use this memory.
Reaction:- Alarm display.
Remedy:You can continue without making a change. The alarm has no negative effects.
To make the change, you have the following possibilities of eliminating the error. Note that removing the access right depends on the current memory configuration of the NCK:
- Try again with a lower value. Observe the change in the value of parameter 3.
- If the control model used allows it, enlarge the memory. Note that this is not possible if parameter 4 equals "iS".
- If permissible, enlarge the NCK user memory. With the appropriate access authorization, you can change the machine data (see above). An increase in buffered memory is generally subject to charge.

Parameters:%1 = Channel number
%2 = Number of the transformations already active
Definitions:Any kinematic transformation in the NCK requires a defined memory space. If MD18866 $MN_MM_NUM_KIN_TRAFOS does not equal zero, it indicates how many kinematic transformations are allowed to be active in the NCK at the same time.
