%2 = Additional information
%3 = Additional information
Definitions:An error occurred during configuration of handling channels. This may be due to one of the following causes:
1 - More axes than allowed have been assigned to both a machining and a handling channel by MD20070 $MC_AXCONF_MACHAX_USED. Additional info1 contains the number of axes affected, additional info2 the maximum number allowed.
2 - In connection with the activation of handling channels, the permissible number of active axes in the system was exceeded. Additional info1 contains the maximum number of axes that can be activated.
3 - A channel was declared as a handling channel and more than the permissible number of axes were activated in it. Additional info1 contains the channel number, additional info2 contains the maximum permissible number of axes.
4 - A channel was declared as a handling channel and more than the permissible number of spindles were activated in it. Additional info1 contains the channel number, additional info2 contains the maximum permissible number of spindles.
5 - The total number of permissible axes was exceeded in the machining channels. Additional info1 contains the maximum total number of axes that can be declared over all machining channels.
Reaction:- NC Start disable in this channel.
- Interface signals are set.
- Alarm display.
Remedy:Please inform the authorized personnel/service department.
1 - In the channel-specific MD20010 $MC_IS_ADD_CHAN, define the channel as a machining channel or remove the corresponding axis from one of the two channels.
2 - In the channel-specific MD20070 $MC_AXCONF_MACHAX_USED, deactivate axes or enable additional handling channels with option.
3 - In the channel-specific MD20010 $MC_IS_ADD_CHAN, define the channel as a machining channel or reduce the number of active axes in the channel.
4 - In the channel-specific MD20010 $MC_IS_ADD_CHAN, define the channel as a machining channel or reduce the number of active spindles in the channel.
5 - In the channel-specific MD20070 $MC_AXCONF_MACHAX_USED, deactivate axes in the respective channels.

10203[Channel %1: ] NC start without reference point (action=%2<ALNX>)
Parameters:%1 = Channel number
%2 = Action number/action name
Definitions:NC start has been activated in the MDI or AUTOMATIC mode and at least one axis that needs to be referenced has not reached its reference point.
Reaction:- Interface signals are set.
- Alarm display.
Remedy:Please inform the authorized personnel/service department. Via the channel-specific MD20700: $MC_REFP_NC_START_LOCK (NC Start without reference point) you can decide whether or not the axis has to be referenced before NC Start. The start of referencing can be enabled channel-specific or axis-specific.
Channel-specific reference point approach: The rising edge of the NC/PLC interface signal <Referenzieren_aktivieren/> (Activate referencing) starts an automatic sequence which starts the axes of the channel in the same sequence as specified in the axis-specific MD34110 $MA_REFP_CYCLE_NR (axis sequence channel-specific referencing). 0: The axis does not participate in channel-specific referencing, but it must be referenced for NC Start, -1: The axis does not participate in channel-specific referencing, but it need not be referenced for NC Start, 1- 8: Starting sequence for the channel-specific referencing (simultaneous start at the same no.), 1 - 31: CPU type
Axis-specific referencing: Press the direction key that corresponds to the approach direction in the axis-specific MD34010 $MA_REFP_CAM_MDIR_IS_MINUS (reference point approach in minus direction).
