Status Synchronized action disabled ($AC_SYNA_STATE)

System variable $AC_SYNA_STATE outputs in bit code whether a synchronized action is disabled via the PLC or a synchronized action.

These bits have the following meaning:

Disabling via PLC or synchronized action have different levels of priority. The following hierarchy of priorities applies:

Priority 1 (highest priority):

Disabled via PLC across all channels (⇒ all synchronized actions in the channel are inhibited)

Priority 2:

Disabled via synchronized action

Priority 3:

Individual disabling via PLC (⇒ a single synchronized action in the channel is disabled)

$AC_SYNA_STATE only outputs the value of the disable with the highest priority, even if disabling is simultaneously active via PLC and Synchronized action:
