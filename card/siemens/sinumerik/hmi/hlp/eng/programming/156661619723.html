Programmable axial acceleration correction (ACC)

In critical program sections, it may be necessary to reduce the maximum possible axis or spindle acceleration temporarily, for example, to prevent mechanical vibrations.

For this purpose, the NC command ACC is available with which acceleration correction can be programmed for each path axis, positioning axis, or spindle.

The correction value is a percentage that refers to the parameterized maximum acceleration of the affected axis/spindle. The product of the correction value and the parameterized maximum acceleration is the effective maximum acceleration of the axis/spindle. For a correction value of 100%, the parameterized maximum acceleration remains unchanged.

The axial acceleration correction can be programmed independently for axis/spindle movements in the NC program and for axis/spindle movements in synchronized actions:

  • If the command is in the NC program, the acceleration correction only affects path axis or positioning axis movements or spindle movements that are programmed in the NC program for this axis/spindle. Movements of the axis/spindle that are programmed in synchronized actions are not corrected.

  • If the command is in a synchronized action, the acceleration correction only affects movements of the axis/spindle that are programmed in synchronized actions. Movements of the axis/spindle that are programmed in the NC program are not corrected.

The axial acceleration correction is effective for all types of interpolation.

Syntax

Activation

ACC[<Ax>]=<value>ACC[SPI(<n>)]=<value>ACC(S<n>)=<value>

Deactivation

ACC[...]=100

Syntax

ACC

Acceleration change for the specified axle/spindle

<Ax>

Channel axis name

SPI(<n>) / S<n>

Spindle identifier

SPI(<n>) and S<n> are identical in terms of function.

<n>

Spindle number

Note:
SPI converts spindle numbers into axis identifiers. The transfer parameter (<n>) must contain a valid spindle number.

<value>

Acceleration change in percent

Type:

INT

Range of values:

1 … 200

Examples

Example 1: Acceleration corrections for axis and spindle movements in the NC program

Program codeComment
... 
N50 ACC[X]=80; The axis slide in the X direction should only be traversed with 80% acceleration.
N60 ACC[SPI(1)]=50; Spindle 1 should only accelerate or brake with 50% of the acceleration capacity.
... 

Example 2: Acceleration correction for path and positioning axis movements in the NC program

Program codeComment
... 
N100 ACC[X]=1 
N110 DO POS[X]=100; ACC not effective.
N120 WAITP(X) 
N130 X=0; ACC effective.
N140 POS[X]=100; ACC effective.
... 

Example 3: Acceleration correction for axis movements in synchronized actions

Program codeComment
... 
N100 DO ACC[X]=1 
N110 DO POS[X]=100; ACC effective.
N120 WAITP(X) 
N130 X=0; ACC not effective.
...
