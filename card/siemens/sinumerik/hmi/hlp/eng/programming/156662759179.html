Deactivating/reactivating the assignment of an interrupt routine (DISABLE, ENABLE)

A SETINT statement can be deactivated with DISABLE and reactivated with ENABLE without losing the input → interrupt routine assignment.

Syntax

DISABLE(<n>)ENABLE(<n>)

Meaning

DISABLE(<n>):

Command: Deactivating the interrupt routine assignment of input <n>

ENABLE(<n>):

Command: Reactivating the interrupt routine assignment of input <n>

<n>:

Parameter: Number of the interrupt signal

Type:

INT

Range of values:

1 ... 32

Example

Program codeComment
N20 SETINT(3) PRIO=1 ABHEB_Z; If input 3 switches, then interrupt ; routine "ABHEB_Z" should start.
... 
N90 DISABLE(3); The SETINT statement from N20 is deactivated.
... 
N130 ENABLE(3); The SETINT statement from N20 is reactivated.
...
