Specification of orientation for two contact points (ORICURVE, ORICURINV, PO[XH]=, PO[YH]=, PO[ZH]=)

Programming the change in orientation using a second curve in space

Another way to program the tool orientation, besides using the tool tip along a curve in space, is to program the motion of a second contact point of the tool using ORICURVE or ORICURINV. In this way, changes in tool orientation can be defined uniquely, as when programming the tool vector itself.

Note

Machine manufacturer

The machine manufacturer's notes regarding the axis identifiers that can be set via machine data for programming the 2nd orientation path of the tool must be observed.

ORICURVE

When interpolating the tool orientation by ORICURVE, the tool orientation is defined by 2 curves in space. The TCP (Tool Center Point) is usually programmed with the geometry axes X, Y, Z. The 2nd curve in space describes the movement of a point on the tool in space and is programmed with the identifiers XH, YH, ZH. The tool orientation is then defined by the following vector Ω:

Ω = (X, Y, Z) - (XH, YH, ZH)

For the coordinates (XH, YH, ZH), polynomials as well as B splines can be programmed.

ORICURINV

For the interpolation of tool orientation at ORICURINV, the tool orientation is also defined by 2 curves in space as for ORICURVE. The TCP is also programmed with the geometry axes X, Y, Z. The 2nd curve in space describes the movement of a point on the tool in space and is also programmed with the identifiers XH, YH, ZH. The tool orientation is then defined by the following vector Σ:

Σ = (XH, YH, ZH) - (X, Y, Z)

Thus, vector Σ has the opposite orientation as vector Ω.

For the coordinates (XH, YH, ZH), polynomials and B splines can also be programmed.

Programming

This type of interpolation can be used to program points (using G1) or polynomials (using POLY) for the two curves in space. Circles and involutes are not permitted. A BSPLINE spline interpolation and the "Combine short spline blocks" function can also be activated.

Other types of splines (ASPLINE and CSPLINE) and the activation of a compressor using COMP... are not permissible.

The motion of the two contact points of the tool can be predefined up to the 5th degree when programming the orientation polynomials for coordinates.

References

For more information about the function "Combine short spine blocks", refer to the

Function Manual Basic Functions.

Syntax

N... ORICURVE

 

N... ORICURINV

 

N... PO[XH]=(xe, x2, x3, x4, x5)

; Indication of the movement of the second

; contact point of the tool and

; additional polynomials of the respective

; coordinates

N... PO[YH]=(ye, y2, y3, y4, y5)

N... PO[ZH]=(ze, z2, z3, z4, z5)

Parameter

ORICURVE

Interpolation of the tool orientation specifying a movement between two contact points of the tool.

ORICURINV

Interpolation of the tool orientation as with ORICURVE, but with reversed orientation.

XH YH ZH

Identifiers of the coordinates of the second contact point of the tool of the additional contour as a curve in space

PO[XH]=(xe, x2, x3, x4, x5)

PO[YH]=(ye, y2, y3, y4, y5)

PO[ZH]=(ze, z2, z3, z4, z5)

Apart from using the appropriate end points, the curves in space can also be programmed using polynomials.

xe, ye, ze

End points of the curve in space

xi, yi, zi

Coefficients of the polynomials up to the 5th degree

Note

Identifiers XH YH ZH for programming a 2nd orientation path

The identifiers must be selected such that no conflict arises with the other identifiers of the linear axes X Y Z, rotary axes, or other interpolation parameters.

Example rotary axes

  • A2 B2 C2 Euler angle or RPY angle

  • A3 B3 C3 direction vectors

  • A4 B4 C4 or A5 B5 C5 surface normal vectors

  • A6 B6 C6 rotation vectors

  • A7 B7 C7 intermediate point coordinates
