Block N/DIAGN: Global diagnostic data

This module contains information about global NC diagnostic data.
Net times: Time without interrupts by higher priority time levels.
Gross times: Time with interrupts by higher priority time levels.
Time levels in order of their priority: position controller, interpolator, block preparation.
 
actCycleTimeBrut Sum of current gross runtime
Column index:
4 		Line index:
See description below 		Max. line index:
(1000 * numCores) + 16
Total of the current gross run times of all channels in ms.

Line index:
Selects a specific SW task:
Line index 1: SERVO
Line index 2: IPO
Line index 3: VL
Line index 4: PLC
Line index 5: SYNACT
Line index 6: COS
Line index 7: DRIVE (low priority)
Line index 8: EXCOM (domain services)
Line index 9: CYCLE (cycl. tasks: SERVO+IPO+Soft-PLC times)
This value refers to the task with the longest cycle time (see $POSCTRL_CYCLE_TIME, $IPO_CYCLE_TIME and $PLC_CYCLE_TIME).
It states how long this task has run within its cycle. The run times of other cyclic tasks are only taken into account if they delay the execution of the considered task due to interruption.
The value indicates the extent to which the set cycle time is used, and whether a level overflow is imminent.
In contrast, the variable actCycleTimeNet refers to the run times of all cyclic tasks. Therefore, it is not unusual if this value is greater than actCycleTimeBrut.
Line index 10: Reserved
Line index 11: INT (compile cycles in the interpreter)
Line index 12: EES (EES - async. - subtask)
Line index 13: PREPJOB (non-cyclic task on an auxiliary core)
Line index 14: PLC start delay of the task from the start of the PLC cycle
Line index 15: PLC task in the SERVO cycle
Line index 16: PLC CYCLE (PLC + PLC task in the SERVO cycle

An offset to the above line index determines the core on which the task runs:
0: Any core
1000: Main core
2000: Additional core 1
3000: Additional core 2
numCores gives the number of cores used by the NCK, and taskAvailable whether the task is present on the core.
