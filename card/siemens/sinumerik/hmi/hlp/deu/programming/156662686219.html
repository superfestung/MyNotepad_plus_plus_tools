Parametrierbarer Programmrücksprung RET(...) / RETB(...)

Im Allgemeinen wird aus einem Unterprogramm in das aufrufende Programm zurückgesprungen. Die Bearbeitung wird dann mit der auf den Unterprogrammaufruf folgenden Programmzeile fortgesetzt.

Daneben gibt es aber auch Anwendungsfälle, für die ein anderes Rücksprungverhalten erforderlich ist, wie z. B.:

  • Fortsetzung der Programmbearbeitung nach Aufruf der Abspanzyklen im ISO-Dialekt-Modus (nach der Konturbeschreibung)

  • Rücksprung ins Hauptprogramm aus einer beliebigen Unterprogrammebene (auch nach ASUP) beim Fehlerhandling

  • Rücksprung über mehrere Programmebenen für spezielle Anwendungen in Compile-Zyklen und im ISO-Dialekt-Modus

In solchen Fällen kommen die vordefinierten Prozeduren RET(...) und RETB(...) zur Anwendung, die Rückspünge über Programmebenen hinweg auf den als Ziel definierten Programmsatz ermöglichen.

Syntax

RET/RETB("<Target>")RET/RETB("<Target>",<NextBlock>)RET/RETB("<Target>",<NextBlock>,<NrRetJumpLevels>)RET/RETB("<Target>", ,<NrRetJumpLevels>)RET/RETB("<Target>",<NextBlock>,<NrRetJumpLevels>,<RetJumpToProgStart>)RET/RETB( , ,<NrRetJumpLevels>,<RetJumpToProgStart>)

Bedeutung

RET(...)

RET(...) sucht das angegebene Rücksprungziel zuerst in Richtung Programmende. Ist die Suche nicht erfolgreich, wird als Nächstes in Richtung Programmanfang gesucht.

RETB(...)

RETB(...) sucht das angegebene Rücksprungziel nur in Richtung Programmanfang.

<Target>

1. Parameter: Rücksprungziel

Gibt den Satz an, mit dem die Programmbearbeitung fortgesetzt werden soll.

Ist der Parameter <NrRetJumpLevels> nicht programmiert, dann befindet sich das Rücksprungziel in dem Programm, aus dem das aktuelle Unterprogramm aufgerufen wurde.

Typ:

STRING

Wert:

Als Rücksprungziel sind folgende Angaben möglich:

  • Satznummer

  • Sprungmarke

  • Zeichenkette wie z. B. Programm- oder Variablenname

    Die Zeichenkette muss folgende Bedingungen erfüllen:

    • Leerzeichen am Ende (im Unterschied zur Sprungmarke, die durch einen ":" am Ende gekennzeichnet ist)

    • Vor der Zeichenkette dürfen nur eine Satznummer und/oder eine Sprungmarke gesetzt sein, keine Programmbefehle.

Hinweis:
Das angegebene Sprungziel muss im Programm, in das zurückgesprungen wird, vorhanden sein. Andernfalls liefert die Suche kein Ergebnis und es wird Alarm 14080 ("Sprungziel nicht gefunden") ausgegeben. Dies ist auch der Fall, wenn sich beim Rücksprung mit RETB das Sprungziel nicht in Richtung Programmanfang, sondern in Richtung Programmende befindet.

<NextBlock>

2. Parameter: Folgesatz

Dieser Parameter ist zu programmieren, wenn die Programmbearbeitung nicht direkt mit dem Zielsatz, sondern erst mit dem Folgesatz fortgesetzt werden soll.

Typ:

INT

Wert:

0

Programmbearbeitung mit dem Satz fortsetzen, der im Parameter <Target> als Rücksprungziel angegeben ist.

> 0

Programmbearbeitung mit dem darauf folgenden Satz fortsetzen.

<NrRetJumpLevels>

3. Parameter: Anzahl der zu überspringenden Programmebenen

Gibt die Anzahl der Programmebenen an, die die überspringen sind, um zur Programmebene mit dem Zielsatz zu gelangen.

Typ:

INT

Wert:

1

Das Programm wird in der "aktuellen Programmebene -1" fortgesetzt (wie RET ohne Parameter).

2

Das Programm wird in der "aktuellen Programmebene -2" fortgesetzt, d. h., es wird eine Ebene übersprungen.

3

Das Programm wird in der "aktuellen Programmebene -3" fortgesetzt, d. h., es werden zwei Ebenen übersprungen.

...

...

Wertebereich:

1 ... 15

<RetJumpToProgStart>

4. Parameter: Rücksprung auf Programmanfang

Dieser Parameter ist zu programmieren, wenn bei einem Rücksprung in das Hauptprogramm und dort aktivem ISO-Dialekt-Modus das Programm am Programmanfang fortgesetzt werden soll.

Typ:

BOOL

Wert:

1

Wenn der Rücksprung ins Hauptprogramm erfolgt und dort ein ISO-Dialekt-Modus aktiv ist, wird auf den Programmanfang verzweigt.

Hinweis

Wird als Rücksprungziel eine Zeichenkette angegeben, die keine Satznummer ist, wird bei der Suche zunächst davon ausgegangen, dass es sich um eine Sprungmarke handelt. Ein als Rücksprungziel verwendeter Programm- oder Variablenname darf daher nicht mit dem Namen einer Sprungmarke übereinstimmen, da sonst der Rücksprung immer auf den Satz mit der Sprungmarke und nicht auf den Satz mit dem Programm- oder Variablennamen ausgeführt wird (siehe Beispiel 2).

Randbedingungen

Rücksprung über mehrere Programmebenen

Beim Rücksprung über mehrere Programmebenen werden die SAVE-Anweisungen der einzelnen Programmebenen ausgewertet.

Ist bei einem Rücksprung über mehrere Programmebenen ein modales Unterprogramm aktiv und ist in einem der übersprungenen Unterprogramme der Abwahlbefehl MCALL für das modale Unterprogramm programmiert, bleibt das modale Unterprogramm weiterhin aktiv.

Achtung

Rücksprung kann Auswirkungen auf modale Einstellungen haben

Bei einem Rücksprung über mehrere Programmebenen liegt es in der alleinigen Verantwortung des Anwenders, dafür zu sorgen, dass mit den erforderlichen modalen Einstellungen fortgesetzt wird. Dies kann z. B. durch Programmierung eines entsprechenden Hauptsatzes erreicht werden.

Abarbeiten von Extern

Werden Haupt- und Unterprogramme über die Funktion "Abarbeiten von Extern" von einem externen Programmspeicher abgearbeitet, muss das mit RET(...) programmierte Rücksprungziel innerhalb des Nachladespeichers liegen. Andernfalls wird das Sprungziel nicht gefunden und es kommt zum Programmabbruch und zur Ausgabe von Alarm 14000 ("Unzulässiges Dateiende").

Hinweis

Um externe Programme ohne Einschränkungen hinsichtlich der programmierten Sprunganweisungen abarbeiten zu können, wird empfohlen, statt der Funktion "Abarbeiten von Extern" die Option "Abarbeiten vom externen Speicher (EES)" zu verwenden.

Beispiele RET(...)

Beispiel 1: Wiederaufsetzen im Hauptprogramm nach ASUP-Bearbeitung

ProgrammierungKommentar
N10010 CALL "UP1"; Programmebene 0 (Hauptprogramm)
  N11000 PROC UP1; Programmebene 1
  N11010 CALL "UP2"
  N12000 PROC UP2; Programmebene 2
  ... 
  N19000 PROC ASUP; Programmebene 3 (ASUP-Bearbeitung)
  ... 
  N19100 RET("N10900", ,$P_STACK); Unterprogrammrücksprung ins Hauptprogramm; $P_STACK: aktuelle Programmebene
N10900; Zielsatz im Hauptprogramm
N10910 MCALL; Modalen Unterprogrammaufruf ausschalten
N10920 G0 G60 G40 M5; Weitere modale Einstellungen initialisieren

Beispiel 2: Zeichenkette als Rücksprungziel

Hauptprogramm:

ProgrammcodeKommentar
PROC MAIN_PROGRAM 
N1000 DEF INT iVar1=1, iVar2=4 
N1010 ... 
N1200 subProg1; Aufruf von Unterprogramm "subProg1"
N1210 M2 S1000 X10 F1000 
N1220 ...... 
N1400 subProg2; Aufruf von Unterprogramm "subProg2"
N1410 M3 S500 Y20 
N1420 .. 
N1500 lab1: iVar1=R10*44 
N1510 F500 X5 
N1520 ... 
N1550 subprog1: G1 X30; "subProg1" ist hier als Sprungmarke definiert.
N1560 ... 
N1600 subProg3; Aufruf von Unterprogramm "subProg3"
N1610 ... 
N1900 M30 

Unterprogramm subProg1:

ProgrammcodeKommentar
PROC subProg1 
N2000 R10=R20+100 
N2010 ... 
N2200 RET("subProg2"); Rücksprung ins Hauptprogramm auf den Satz N1400

Unterprogramm subProg2:

ProgrammcodeKommentar
PROC subProg2 
N2000 R10=R20+100 
N2010 ... 
N2200 RET("iVar1"); Rücksprung ins Hauptprogramm auf den Satz N1500

Unterprogramm subProg3:

ProgrammcodeKommentar
PROC subProg3 
N2000 R10=R20+100 
N2010 ... 
N2200 RET("subProg1"); Rücksprung ins Hauptprogramm auf den Satz N1550
