Programmrücksprung RET

RET bewirkt wie M17/M30 den Rücksprung in das aufrufende Programm auf den Satz nach dem Unterprogrammaufruf. Anders als bei M17/M30 wird ein im Kanal aktiver Bahnsteuerbetrieb aber nicht unterbrochen.

Durch die Programmierung von Parametern kann das Rücksprungverhalten von RET angepasst werden (siehe "").

Hinweis

RET kann nur in Unterprogrammen verwendet werden, die nicht mit dem Attribut SAVE definiert wurden.

Syntax

RET muss in einem eigenen Satz stehen:

PROC <ProgName>
...
RET

Beispiel

Hauptprogramm

ProgrammcodeKommentar
PROC MAIN_PROGRAM; Programmanfang
... 
N50 SUB_PROG; Unterprogrammaufruf: SUB_PROG
N60 ... 
... 
N100 M30; Programmende

Unterprogramm

ProgrammcodeKommentar
PROC SUB_PROG 
... 
N100 RET; Rücksprung erfolgt auf Satz N60 im Hauptprogramm.
