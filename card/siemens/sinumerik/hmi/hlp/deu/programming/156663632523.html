Adaption ein-/ausschalten (CADAPTON, CADAPTOF)

Mit den vordefinierten Prozeduren CADAPTON() und CADAPTOF() können vordefinierte Adaptionen zur Anpassung von Dynamik- oder Regelungsparametern aus dem Teileprogramm heraus aktiviert, aktualisiert und deaktiviert werden. Verwendung finden sie auch im CYCLE782.

Syntax

... 
CADAPTON(<Result>,<Axis>,<InVar>[,<InVal>]) 
... 
CADAPTOF(<Result>,<Axis>,<InVar>) 
... 

Bedeutung

CADAPTON():

Adaptionsbeziehung aktivieren

CADAPTOF():

Adaptionsbeziehung deaktivieren

Hinweis:
Permanent wirksame Adaptionsbeziehungen (MD16501 = 1) ignorieren CADAPTOF(), ein ggf. zuvor programmierter Eingangswert (<InVal>) bleibt wirksam.

<Result>:

Ergebnisvariable: Rückgabewert für den Status (Call by reference-Parameter)

Datentyp:

INT

Wert:

0

kein Fehler

1

keine gültige Adaptionstabelle parametriert

2

Parameter <Axis> ungültig

3

Parameter <InVar> ungültig

4

reserviert

5

Parameter <InVal> ungültig

<Axis>:

Maschinenachsname der Eingangsachse der Adaptionsbeziehung

Datentyp:

AXIS

Wertebereich:

Im Kanal definierte Maschinenachsnamen

Hinweis:
Mit diesem Parameter werden diejenigen Adaptionen adressiert, die in MD16504 $MN_CADAPT_INPUT_AX einen dem Parameter <Axis> entsprechenden Wert eingetragen haben. Die weiteren Parameter <InVar> und <InVal> sind dieser Achse zugeordnet.

<InVar>:

Eingangsgröße der Adaptionsbeziehung

Datentyp:

INT

Wert:

1

Trägheit der Achse

2

Achsposition

3

Achsgeschwindigkeit

<InVal>:

Eingangswert der Adaptionsbeziehung

Optionaler Parameter für die intelligente Lastanpassung (<InVar>=1).

Datentyp:

REAL

Wert:

> 0

Aktuelle Trägheit

= 0

Last/Beladung ist undefiniert bzw. nicht bekannt. Die zugeordneten Adaptionen liefern in diesem Fall als Ausgangsgröße den Ersatzwert 1.0.

< 0

Der Aufruf wird mit <Status> = 5  (Parameter <InVal> ungültig) beendet. Der zuletzt aktivierte Eingangswert bleibt weiterhin aktiv.

Hinweis:
Ohne Programmierung eines Eingangswerts wird der zuletzt programmierte Wert bzw. nach einem Steuerungshochlauf der Standardwert (= 0) wirksam.

Beispiel

ProgrammcodeKommentar
DEF INT RESULT; Definition der Ergebnisvariablen
... 
CADAPTON(RESULT,MX1,2); Adaption aktivieren
IF RESULT <> 0; Auswertung der Ergebnisvariablen (nach jeder CADAPTON/CADAPTOF-Anweisung erforderlich)
MSG("CADAPT-RESULT=" << RESULT) 
STOPRE 
SETAL(61000) 
ENDIF 
... 
CADAPTOF(RESULT,MX1,2); Adaption deaktivieren
...; Auswertung der Ergebnisvariablen
...
