Baustein N/DIAGN: Globale Diagnosedaten

Dieser Baustein enthält Informationen über globale Diagnosedaten der NC.
Nettozeiten: Zeit ohne Unterbrechungen durch höherpriore Zeitebenen
Bruttozeiten: Zeit mit Unterbrechungen durch höherpriore Zeitebenen
Zeitebenen in der Reihenfolge ihrer Priorität: Lageregler, Interpolator, Satzaufbereitung
 
actCycleTimeBrut Summe der aktuellen Bruttolaufzeit
Spaltenindex:
4 		Zeilenindex:
Siehe Beschreibung unten 		Max. Zeilenindex:
(1000 * numCores) + 16
Summe der aktuellen Bruttolaufzeit aller Kanäle in ms.

Zeilenindex:
Wählt eine bestimmte SW-Task aus:
Zeilenindex 1: SERVO
Zeilenindex 2: IPO
Zeilenindex 3: VL
Zeilenindex 4: PLC
Zeilenindex 5: SYNACT
Zeilenindex 6: COS
Zeilenindex 7: DRIVE (niederprior)
Zeilenindex 8: EXCOM (Domaindienste)
Zeilenindex 9: CYCLE (zykl. Tasks: SERVO+IPO+Soft-PLC Zeiten)
Dieser Wert bezieht sich auf diejenige Task mit der längsten Zyklus-Zeit (siehe $POSCTRL_CYCLE_TIME, $IPO_CYCLE_TIME und $PLC_CYCLE_TIME).
Sie gibt an, wie lange diese Task innerhalb ihres Zykluses gelaufen ist. Die Laufzeiten anderer zyklischen Tasks werden nur insofern berücksichtigt, wie sie die Ausführung der betrachteten Task durch Unterbrechung verzögern.
Der Wert gibt Aufschluss darüber, in welchem Maße die eingestellte Zykluszeit ausgenutzt wird und ob ein Ebenenüberlauf droht.
Im Gegensatz dazu, bezieht sich die Variable actCycleTimeNet auf die Laufzeiten aller zyklischen Tasks. Es ist daher nicht ungewöhnlich, wenn dieser Wert größer ist als actCycleTimeBrut.
Zeilenindex 10: reserviert
Zeilenindex 11: INT (Compilezyklen im Interpreter)
Zeilenindex 12: EES (EES - Async - Subtask)
Zeilenindex 13: PREPJOB (nichtzykl. Task auf einem Nebencore)
Zeilenindex 14: PLC Startverzögerung der Task vom Beginn des PLC-Taktes
Zeilenindex 15: PLC Task im SERVO-Takt
Zeilenindex 16: PLC CYCLE (PLC + PLC Task im SERVO-Takt

Ein Offset zum obigen Zeilenindex bestimmt den Core, auf dem die Task läuft:
0: beliebiger Core
1000: Main-Core
2000: Additional Core 1
3000: Additional Core 2
numCores gibt die Anzahl der vom NCK genutzen Cores an und taskAvailable ob die Task auf dem Core vorhanden ist.
Typ
TYPE_DOUBLE 		Einheit
Zeit (ms) 		Standardwert
0 		Minimalwert
0 		Maximalwert
