Lokale Benutzergruppen anlegen

Einleitung

Wenn Sie eine neue lokale Benutzergruppe anlegen möchten, können Sie den Dialog "Lokale Benutzergruppe anlegen" nutzen.

Voraussetzung

  • Sie verfügen über die Rolle "Security Admin Hersteller" oder "Security Admin Anwender".

  • Die aktuelle Domäne ist nicht mit einer zentralen Benutzerverwaltung verbunden.

Achtung

Lokale Benutzergruppen werden gelöscht beim Verbinden von zentraler Benutzerverwaltung

Wenn eine vorher lokal angelegte Benutzergruppe eine namentliche Entsprechung auf der Domäne der zentralen Benutzererwaltung besitzt, gibt diese lokale Benutzergruppe den Benutzern aus der zentralen Benutzergruppe ihrer Rechte über die Maschinenrollen.

Deswegen werden alle lokalen Benutzergruppen gelöscht, die keine namentliche Entsprechung auf der Domäne der zentralen Benutzerverwaltung besitzen, sobald Sie ihre aktive Domäne mit einer zentralen Benutzerverwaltung verbinden.

  • Wenn Sie neben den Standard Security Admins der Domäne noch weitere Security Admins anlegen, empfehlen wir, dass Sie diesen Benutzer direkt mit der Machinenrolle "Security Admin" verknüpfen.

Vorgehensweise

  1. Im Bedienbereich "Inbetriebnahme" rufen Sie die Benutzerverwaltung über die Softkeys "Security > Benutzerverwaltung" auf.

  2. Klicken Sie auf den Softkey "Benutzergruppen". Klicken Sie auf der Übersichtsseite auf "+ Benutzergruppe", um eine neue lokale Benutzergruppe anzulegen. Ein Pop-up Dialog öffnet sich.


    Neue lokale Benutzergruppe anlegen

  3. Vergeben Sie einen beliebigen Namen für die lokale Benutzergruppe und eine möglichst treffende Beschreibung, welche die Benutzer dieser Gruppe verbindet.

    Hinweis

    Beachten Sie folgende Hinweise bei der Namenswahl:

    • Sonderzeichen sind bei der Vergabe des Namens der lokalen Benutzergruppe nicht erlaubt.

    • In der Hersteller-Domäne dürfen die Benutzergruppen nicht den gleichen Namen haben wie Maschinenrollen.
    Hinweis

    Beachten Sie, dass in der Hersteller-Domäne die Benutzergruppen nicht den gleichen Namen haben dürfen wie die Maschinenrollen.
    Hinweis

    Beachten Sie, dass bei der Vergabe des Namens für die lokale Benutzergruppe Sonderzeichen nicht erlaubt sind.

  4. Wenn Sie bereits Maschinenrollen angelegt haben, wählen Sie eine oder mehrere Maschinenrolle(n) aus, welcher die Benutzergruppe zugeordnet werden soll.
    Die Mitglieder dieser Benutzergruppe bekommen den höchsten AccessLevel dieser Maschinenrolle(n). Falls der Benutzergruppe keine Maschinenrollen zugewiesen werden, haben die Mitglieder der Benutzergruppe keine Rechte (Keyswitch0).

  5. Wenn Sie bereits Benutzer angelegt haben, wählen Sie Benutzer aus, die Mitglied dieser lokalen Benutzergruppe sind.

  6. Klicken Sie auf "OK", um die neue lokale Benutzergruppe in die Liste der Benutzergruppen zu übernehmen. Legen Sie bei Bedarf weitere lokale Benutzergruppen an, indem Sie die Schritte 2-6 wiederholen.
