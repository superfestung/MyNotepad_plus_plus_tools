Archiv auf SINUMERIK ONE übertragen

Sie haben die Möglichkeit, Archive im Format *.arc oder *.dsf in die SINUMERIK ONE-Umgebung zu übertragen.

Hinweis

Wenn Sie eine virtuelle Maschine verwenden, muss das Archiv von einer realen Ablage auf das Speichermedium der virtuellen Maschine übertragen werden.
Hinweis

Sie können ein SINUMERIK 840D sl-Archiv in SINUMERIK ONE übertragen. Erstellen Sie auf der SINUMERIK 840D sl-Steuerung ein Inbetriebnahmearchiv wie gewohnt im Format *.arc.

SINUMERIK 840D sl-Inbetriebnahmearchive sind vom Typ Backup und enthalten maschinenindividuelle Daten. Beim Übertragen auf SINUMERIK ONE werden die vorhandenen Daten überschrieben, z. B. basesys.ini. Inkompatible Einstellungen zwischen den Steuerungssystemen werden ignoriert, z. B. mmc.ini und systemconfiguration.ini.

Voraussetzung

  • Zum Einlesen eines übertragenen Backups oder Setups ist die Zugriffsstufe "Hersteller" erforderlich.

Hinweis

HMI-Komponente

Ein DSF-Archiv mit HMI-Komponente einer virtuellen Maschine ist nicht kompatibel mit einer realen Maschine und umgekehrt. Es werden nur die PLC-Alarmsystemtexte im Format *.xml und *.ts aus dem Archiv übernommen.

Falls erforderlich, importieren Sie eigene SINUMERIK ONE-Einstellungen manuell. Kopieren Sie z. B. Systemtexte in das Verzeichnis
.../oem/sinumerik/hmi/Ing.

Hinweis

PLC-Komponente

Wenn das DSF-Archiv im TIA Portal erstellt wurde, übernimmt SINUMERIK ONE auch eine vorhandene PLC-Komponente. Folgende Voraussetzungen gelten:

  • Der im TIA Portal eingerichtete NCU-Typ stimmt mit dem Typ der Zielsteuerung überein, z. B. NCU 1750.

  • Die im TIA Portal eingerichtete CNC-Software ist mit dem Stand der Zielsteuerung kompatibel.

Hinweis

Security-Komponente

Wenn das DSF-Archiv aus einem anderen System eine Security-Komponente enthält, sind die PLC-Daten vertraulich und somit kennwortgeschützt. Zur Übertragung benötigen Sie das Security-Kennwort des Archivs.

Geben Sie das Security-Kennwort im Dialog vor der Archivübertragung oder im Bedienbereich
"Inbetriebnahme > Security > Security-Einstellungen" ein.

Hinweis

Verhindern Sie eine unabsichtliche Archivverwechslung bei der Übertragung mehrerer Archive.

Es wird empfohlen, für jedes System im gleichen Firmennetzwerk ein individuelles VNC-Kennwort einzurichten. Damit vermeiden Sie, dass ein DSF-Archiv mit Security-Komponente unabsichtlich an der falschen Maschine eingelesen wird.
