Security Eventlog in SINUMERIK Operate anzeigen

Einleitung

Die Security-relevanten Ereignisse werden in SINUMERIK Operate unter "Eventlog" angezeigt. Hier können Sie gezielt nach Schlagworten suchen, die im Eventlog protokolliert werden.

Der Verbindungsstatus wird unter "Server-Verbindung" angezeigt. Wenn keine Verbindung zum Syslog-Server besteht, wird unter "Server-Verbindung" angezeigt, dass die ausstehenden Eventlog-Ereignisse an den Server gesendet werden müssen.

Voraussetzung

  • Mit aktiver Benutzerverwaltung: Sie verfügen über die Rolle "Security Admin" oder die Zugriffsstufe 2 (Service).

ODER

  • Ohne aktive Benutzerverwaltung: Sie verfügen mindestens über die Zugriffsstufe 3 (Anwender).

Vorgehensweise

  1. Wählen Sie den Bedienbereich "Inbetriebnahme" an.

  2. Drücken Sie die Softkeys "Security > Security Eventlog > Eventlog".

    Die Security-relevanten Ereignisse werden angezeigt.

    Hinweis

    Wenn neue Security-relevanten Ereignisse hinzukommen, wird die Anzeige nicht automatisch aktualisiert. Drücken Sie auf die Schaltfläche "Aktualisieren", um die neuesten Security-relevanten Ereignisse anzuzeigen.

    Neue Security-relevanten Ereignisse erscheinen im Eventlog ganz unten.

  3. Drücken Sie die Schaltfläche "Suchen", wenn Sie im Eventlog gezielt nach Schlagworten suchen wollen.

  4. Geben Sie in der Suchmaske den gewünschten Begriff ein, z. B. "USB", und bestätigen Sie mit "OK".
    Der Cursor wird auf den ersten Eintrag gesetzt, der dem Suchbegriff entspricht.

  5. Wenn es sich beim gefundenen Text nicht um die gesuchte Textstelle handelt, drücken Sie die Schaltfläche "Weitersuchen".

  6. Drücken Sie die Schaltfläche "Abbruch", um den Suchvorgang abzubrechen.

  7. Drücken Sie die Schaltfläche "Aktualisieren", um die Liste der angezeigten Treffer zu aktualisieren.

  8. Um alle lokalen Einträge im Security Eventlog zu löschen, drücken Sie die Schaltfläche "Alles löschen".

    Bei einem verteilten System werden die Security-relevanten Ereignisse von der NCU und vom IPC gelöscht, wenn die NCU und der IPC gekoppelt sind. Wenn die NCU nicht mit dem IPC gekoppelt ist und der Löschvorgang auf dem IPC angestoßen wurde, werden die Security-relevanten Ereignisse nur vom IPC gelöscht.
    Alle Einträge, die an den Server gesendet wurden, bleiben bestehen.
Hinweis

Um die Schaltfläche "Alles löschen" anzeigen und bedienen zu können, müssen Sie die folgenden Zugriffsstufen/Rollen haben:

  • Bei aktivierter Benutzerverwaltung: Rolle "Security Admin"

  • Bei nicht aktivierter Benutzerverwaltung: Zugriffsstufe 3 (Anwender)
