NCU Kopplungszertifkate können nicht erreicht werden

Fehlermeldung in SINUMERIK Operate

  • Prüfen Sie, ob die Verbindung zur NCU korrekt ist.

  • Wenn die Verbindung korrekt ist und der Fehler weiter besteht, kann die NCU defekt sein. Setzen Sie die NCU auf Werkseinstellungen zurück.

NCU kann Kopplungszertifkate nicht erstellen

Kopplungsstatus in SINUMERIK Operate zeigt "Nicht im Kopplungsmodus"

  • Status wird angezeigt, wenn der installierte Softwarestand der NCU zu alt ist, sodass das Koppeln als Funktion nicht durchgeführt werden kann. Aktualisieren Sie den Softwarestand der NCU.

  • Wenn der Softwarestand der NCU aktuell ist, kann der Security Host abgestürzt oder nicht gestartet worden sein. Starten Sie den Security Host manuell über den Befehl "sc start sec".

NCU wurde bereits mit anderem IPC gekoppelt. Dies ist nicht der IPC, mit dem sie verbunden ist. Koppeln kann nicht durchgeführt werden.

Kopplungsstatus in SINUMERIK Operate zeigt "Bereits mit anderem IPC gekoppelt"

  • Verbinden Sie die NCU mit dem ursprünglich gekoppelten IPC

  • Oder setzen Sie die NCU auf Werkseinstellungen zurück und koppeln Sie diese mit dem aktuell verbundenen IPC.

Das Datum auf IPC und NCU stimmt nicht überein. Koppeln kann nicht durchgeführt werden.

Fehlermeldung in SINUMERIK Operate

  • Synchronisieren Sie die Daten von IPC und NCU. Das Datum der NCU können Sie in SINUMERIK Operate über "Inbetriebnahme > HMI > Datum Uhrzeit" einstellen.

Verbindungsprobleme zwischen IPC und NCU. Koppeln oder Entkoppeln können nicht durchgeführt werden.

Fehlermeldung in SINUMERIK Operate

  • Prüfen Sie die korrekte Verbindung von IPC und NCU

  • Führen Sie das Koppeln oder Entkoppeln erneut aus.

Das Hersteller-Passwort zur Authentifizierung ist nicht korrekt.

Fehlermeldung in SINUMERIK Operate

  • Geben Sie das korrekte Hersteller-Passwort ein.
