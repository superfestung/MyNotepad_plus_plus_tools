Richtlinien

Einleitung

Wenn Sie Security-Richtlinien für die Benutzerverwaltung festlegen bzw. anpassen möchten, können Sie dies über den Dialog "Richtlinien" vornehmen. Dort können Sie die Passwortrichtlinien und Anmelderichtlinien für alle Benutzer festlegen bzw. ändern.

Achtung

Daten-Missbrauch

Um Daten-Missbrauch vorzubeugen und ihre Steuerung unter Einsatz der Benutzerverwaltung möglichst sicher zu machen, empfehlen wir Ihnen, hohe Standards für den Einsatz von sicheren Passwörtern anzusetzen.

  • Tipps für die Vergabe von sicheren Passwörtern finden Sie z. B. beim .
Hinweis

Asiatische Schriftzeichen zählen nicht als Groß- oder Kleinbuchstaben

Achten Sie bei der Festlegung der Passwortrichtlinien darauf, dass asiatische Schriftzeichen nicht als Groß- oder Kleinbuchstaben gewertet werden.

  • Legen Sie deswegen in diesen Kategorien keine Beschränkungen fest, wenn das Passwort potenziell asiatische Schriftzeichen enthalten kann.

Passwortrichtlinien ändern

  1. Im Bedienbereich "Inbetriebnahme" rufen Sie die Benutzerverwaltung über die Softkeys "Security > Benutzerverwaltung" auf.

  2. Klicken Sie auf den Softkey "Richtlinien". Sie sehen die aktuellen Einstellungen für die Passwortrichtlinien.


    Richtlinien

  3. Klicken Sie auf die Schaltfläche "Ändern". Sie können die angezeigten Werte nun ändern.

  4. Sie können folgende Einstellungen festlegen:

    • Individuelle Passwortrichtlinien: Die möglichen Einstellungen entnehmen Sie aus obiger Tabelle.

    • Option, dass nach Änderung des Passworts, die letzten X Passwörter nicht genutzt werden dürfen. Dies zwingt den Benutzer dazu, möglichst immer ein neues Passwort zu definieren und keine alten Passwörter wiederzuverwenden. Diese Option kann aktiviert oder deaktiviert werden.

  5. Bestätigen Sie ihre Einstellungen mit "OK".

Anmelderichtlinien ändern

  1. Im Bedienbereich "Inbetriebnahme" rufen Sie die Benutzerverwaltung über die Softkeys "Security > Benutzerverwaltung" auf.

  2. Klicken Sie auf den Softkey "Richtlinien". Sie sehen die aktuellen Einstellungen für die Anmelderichtlinien.

  3. Klicken Sie auf die Schaltfläche "Ändern". Sie können die angezeigten Werte nun ändern.

  4. Sie können folgende Einstellungen festlegen:

    • Option, die Anzahl der ungültigen Anmeldeversuche einzuschränken. Wenn Sie die Anzahl der ungültigen Anmeldeversuche beschränken und keine zusätzliche Frist zum Rücksetzen definiert haben, wird der Benutzer nach der definierten Anzahl von Fehlversuchen sofort gesperrt.

    • Sie können auch eine Sperrzeit festlegen, wie lange der Benutzer nach einem ungültigen Anmeldeversuch warten muss, bevor eine erneute Anmeldung wieder möglich ist.

    • Zusätzlich können Sie auch nach einer definierten Frist, die Anmeldeversuche automatisch rücksetzen lassen. Erst wenn die Frist abgelaufen ist, wird die Anzahl der ungültigen Anmeldeversuche um 1 dekrementiert. Vorausgesetzt, die maximale Anzahl der Anmeldeversuche wurde nicht schon vorher erreicht und das Benutzer-Passwort ist bereits ungültig geworden. Mit dieser Einstellung können Sie die Option „der maximalen Anmeldeversuche“ zusätzlich beeinflussen.

    • Option, einen so genannten Systemnutzungshinweis in der Anmelde-Maske einblenden zu lassen. Der Systemnutzungshinweis wird in Form einer Sprachdatei im Ordner "/user/sinumerik/hmi/lng" pro Sprache hinterlegt.

  5. Bestätigen Sie ihre Einstellungen mit "OK".

Systemnutzungshinweis

Sie haben optional die Möglichkeit, im oberen Bereich des Anmelde-Dialogs der Benutzerverwaltung einen Systemnutzungshinweis einzublenden. Der Systemnutzungshinweis kann zusätzlicher Hinweise, Nutzungsbeschränkungen oder Zustimmungserklärung enthalten, dem alle Benutzer zustimmen müssen, die sich in SINUMERIK Operate über die Benutzerverwaltung anmelden.

  • Der Systemnutzungshinweis wird in Form einer Sprachdatei im Ordner "/user/sinumerik/hmi/lng" pro Sprache hinterlegt.

    Z. B. Benennung der Datei für die englische Sprache: "system_use_notif_eng.ts".

Im folgenden Beispiel sehen sie den Aufbau der Datei:

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE TS>
<TS>
<context>
<name>sysusenotif</name>
<message>
<source>SYS_USE_NOTIF_TEXT</source>
<translation>This is an example for a system use notification with an example text</translation>
</message>
</context>
</TS>
