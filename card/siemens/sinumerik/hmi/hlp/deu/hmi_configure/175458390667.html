Visualisierung von geladenen Trace-Konfigurationen

Wenn eine Trace-Sitzungsdatei geladen wird, die in dieser Anwendung erstellt wurde, werden Signaldaten wie z.  B.:

  • Bezeichner der Achs- und Kanalparameter von Signalen,

  • Bezeichner der Antriebsobjekte,

  • Physikalische Einheiten, Datentypen, Array-Indizes, usw.,

in der Signaltabelle so angezeigt, wie sie zum Zeitpunkt der Messung waren, als die Sitzungsdatei gespeichert wurde.

Die Daten können auch dann geladen und korrekt angezeigt werden, wenn die Bedienoberfläche offline, mit einer anderen Maschine verbunden ist oder wenn sich die Maschinenkonfiguration geändert hat und die geladenen Daten nicht mehr gültig sind.

Die Beschreibungstexte von NC-Variablen werden in der aktuell gewählten Sprache angezeigt, auch wenn die Sitzungsdatei mit einer anderen Spracheinstellung gespeichert wurde.

Sitzungsdateien, die in dieser Trace-Anwendung gespeichert werden, sind mit einem Überprüfungscode geschützt, mit dem festgestellt werden kann, ob der Inhalt der Datei außerhalb der Anwendung geändert wurde. Wenn beim Öffnen einer Datei eine externe Änderung festgestellt wird, wird eine Fehlermeldung angezeigt, und die Sitzungsdatei kann nicht geladen werden.

Kompatibilitätsmodus

Die Trace-Anwendung kann Sitzungsdateien, die in der SINUMERIK Operate-Funktion Trace Klassik oder in SinuCom NC erstellt wurden, im Kompatibilitätsmodus laden, was in der Titelleiste der Trace-Oberfläche angezeigt wird. In diesem Modus ist die Trace-Konfiguration schreibgeschützt. Sie können weder Variablen hinzufügen oder löschen noch die Parameter der Variablen ändern. Nur Visualisierungseinstellungen wie Farben, Linienarten, usw. lassen sich bearbeiten.

Sitzungsdateien, die von den oben genannten Anwendungen gespeichert werden, enthalten nicht alle Informationen über die Signaleigenschaften zum Zeitpunkt der Messung. Daher kann nicht garantiert werden, dass alle in der Signaltabelle angezeigten Daten korrekt sind.

Hinweis

Wenn eine Sitzungsdatei im Kompatibilitätsmodus geladen wird, werden die meisten Signaleigenschaften, z. B. Achs- und Kanalbezeichner, physikalische Einheiten, GUD-Definitionen, usw., aus der aktuellen Maschinenkonfiguration ermittelt.

Die angezeigten Informationen sind nur dann korrekt, wenn die Sitzungsdatei auf die selbe Maschine geladen wird, auf der sie erstellt wurde und sich die Geräteeinstellungen nicht geändert haben.

Die Datentypen der NC-Variablen werden so angezeigt, wie sie aus der Sitzungsdatei geladen wurden. Enthält die Sitzung hingegen PLC-Speicheradressen oder 840D/828D-PLC-Variablen, werden deren Datentypen entsprechend der aktuellen Konfiguration neu ermittelt. Eine Popup-Meldung wird angezeigt, wenn die Anwendung den Datentyp einer PLC-Variablen geändert hat.

Wenn Sie die Konfiguration aus dieser Anwendung erneut speichern, verbleibt sie im Kompatibilitätsmodus.

Um eine bearbeitbare Trace-Konfiguration zu erstellen, starten Sie eine neue Trace-Sitzung.
