Applikationen

Die Applikationen, die in einer Display-Konfigurationen verwendet werden, müssen in der Datei systemconfiguration.ini angegeben werden.

Es werden zwei Typen von Applikationen unterschieden:

  • Typ A: Auf Basis der Klasse SlGfwSideScreenDialog erstellte Applikationen

  • Typ B: Standard-Windows-Applikationen (.EXE)

Alle Applikationen werden beim Systemstart vom Display-Manager automatisch gestartet und beim Herunterfahren des Systems wieder beendet.

Applikationen vom Typ A werden in der Datei systemconfiguration.ini im Abschnitt [dialogs] im Nummernbereich 500 - 999 (OEM-Bereich) deklariert.

Applikationen vom Typ B werden wie OEM-Frame-Applikationen behandelt und werden dementsprechend im Abschnitt [processes] deklariert. Zusätzlich muss für diese Applikationen in der Datei
C:\Program Files (x86)\Siemens\MotionControl\compat\user\OEMFRAME.INI
der Parameter nPlacementMode auf 3 gesetzt werden.

Folgende Applikationen sind im Standard enthalten:

  • PDF-Viewer

  • VNC-Viewer

  • Virtuelle Tastatur

  • Virtuelle MCP

  • Applikation zur Anzeige von Widgets

  • Kameraanzeige

  • WEB Browser

  • SINUMERIK Operate

PDF-Viewer

Für die Projektierung im Standard wird in der Datei systemconfiguration.ini folgendes deklariert:

[dialogs]DLG107= name:=SlPdfApp, implementation:=sldmpdfviewerapp.SlDmPdfViewerApp, process:=SlHmiHost1, preload:=false cmdline:="-pdfFile gestenbedienung.pdf -matteColor #3b4c58"

Für die Projektierung innerhalb einer Display-Konfiguration zu verwendender Applikationsname:

SlPdfApp

Das anzuzeigende Dokument wird im Parameter "cmdline" angegeben. Die beiden Parameter haben folgende Bedeutung:

  • pdfFile: Im Pdf-Viewer anzuzeigende Datei. Die Datei kann sich in den Verzeichnissen appl oder hlp befinden.

  • matteColor: Hintergrundfarbe des PDF-Viewers (Farbe ist hexcodiert anzugeben).

Es können innerhalb einer Display-Konfiguration weitere Instanzen des PDF-Viewers deklariert werden. Der Parameter "name" erhält dabei den Namen der weiteren Instanz und im Parameter "cmdline" wird das von dieser Instanz anzuzeigende Dokument angegeben, z. B.:

DLGxxx= name:=anotherPdf, implementation:=sldmpdfviewerapp.SlDmPdfViewerApp, process:=SlHmiHost1, preload:=false cmdline:="-pdfFile mydocument.pdf -matteColor #3b4c58"

Der PDF-Viewer unterstützt Sprachumschaltung.

VNC-Viewer

Für die Projektierung im Standard wird in der Datei systemconfiguration.ini folgendes deklariert:

[dialogs]DLG108= name:=SlVncApp, implementation:=sldmvncviewerapp.SlDmVncViewerApp, process:=SlHmiHost1, preload:=false, cmdline:="-configuration MyPC"

Für die Projektierung innerhalb einer Display-Konfiguration zu verwendender Applikationsname:

SlVncApp

Die Konfiguration des VNC-Viewers wird in der Datei slvncconfig.ini vorgenommen. Der Name der Sektion, in der die Konfiguration für eine konkrete VNC-Viewer-Instanz erfolgt, wird im Parameter "cmdline" als Wert für das Argument "-configuration" angegeben - im obigem Beispiel "MyPC".

Folgende Parameter sind für jede VNC-Viewer-Instanz anzugeben:

  • host: IP-Adresse oder DNS-Name des Hosts auf dem der zugehörige VNC-Server läuft

  • port: TCP-Port, den der Server verwendet

  • password: Passwort bei entsprechend geschütztem Zugang zum VNC-Server (optional)

  • scalemode: Darstellung Bildinhalt, Skalierungsverhalten

    =0 - (Standard) An Fenster anpassen

    =1 - An Fenster anpassen und Seitenverhältnis beachten

    =2 - Originalgröße verwenden und zentriert anzeigen

    =3 - Originalgröße verwenden; falls zu groß an Fenster anpassen

  • showInfo: Anzeige des Verbindungsstatus

    false - (Standard) Keine Anzeige

    true - Verbindungsstatus wird angezeigt (siehe Parameter showinfoPos bzgl. Position)

  • showInfoPos: Anzeigeposition des Verbindungstatus (nur wirksam mit Parameter showInfo=true)

    =64 - (Standard) Verbindungsstatus wird im VNC-Bild unten angezeigt

    =32 - Anzeige oben

  • scaleHighQuality: Skalierung mit hoher Qualität

    false - (Standard) Normale Skalierungsqualität; kompatibel zum bisherigen Verhalten

    true - Skalierung in hoher Qualität

    Achtung

    Systemauslastung

    Eine hohe Skalierungsqualität benötigt erhöhte Rechenkapazität. Die Systemleistung kann beeinträchtigt werden.

  • showHeaderPanel: Anzeigeleiste für Bedienbereich und Meldezeile (optional, wenn VNC-Area/Bedienbereich projektiert ist)

    false - ausblenden

    true - (Standard) anzeigen

  • viewerRect: Definition des angezeigten VNC-Bildauschnitts

    (Standard) Vollbild

    Durch Komma getrennte Festlegung des angezeigten Bereichs; links, oben, Breite, Höhe (z. B. viewerRect=0,0,800,600)

  • touchSupport: Touch-Unterstützung

    =0 - Single-Touch

    =1 - (Standard) Multi-Touch SINUMERIK VNC-Server

    =2 - Multi-Touch UltraVNC-Server

Hinweis

Alle mit Standard gekennzeichneten Parameter-Einstellungen können auch weggelassen werden.

Beispiel

Für Projektieung in der Datei slvncconfig.ini:

[MyPC]host=192.168.1.183port=5900password=mypasswdscalemode=2showInfo=trueshowInfoPos=32showHeaderPanel=falseviewerRect=10,10,300,300

Virtuelle Tastatur

Für die Projektierung im Standard wird in der Datei systemconfiguration.ini folgendes deklariert:

[dialogs]DLG111= name:=SlKeyboardApp, implementation:=sldmvirtualkeyboardapp.SlDmVirtualKeyboardApp, process:=SlHmiHost1, preload:=false, cmdline:="-settingsFile sldm_keyboard.ini"

Für die Projektierung innerhalb einer Display-Konfiguration zu verwendender Applikationsname:

SlKeyboardApp

Die Applikation darf innerhalb einer Display-Konfiguration nur in einer Instanz verwendet werden.

Im Standard sind drei Größen für die virtuelle Tastatur vorkonfiguriert (siehe Datei sldm_keyboard.ini):

[760x505]Num= KeyboardLayout:="numpadlayout", x:=430, y:=270Alpha= KeyboardLayout:="nclayout", x:=92, y:=50KeyHeight=47[1024x242]Num= KeyboardLayout:="numpadlayout", x:=784, y:=25Alpha= KeyboardLayout:="nclayout", x:=206, y:=25KeyHeight=47[824x210]Num= KeyboardLayout:="numpadlayout", x:=584, y:=22Alpha= KeyboardLayout:="nclayout", x:=5, y:=22KeyHeight=47

Der Abschnittsname entspricht der jeweiligen Framegröße (Angabe in Pixel). Die Attribute haben folgende Bedeutung:

Property

Bedeutung

KeyboardLayout

Layout der virtuellen Tastatur. Zwei Layouts stehen zur Verfügung:

  • numpadlayout: Umfasst nur Tasten zur Eingabe von Zahlen

  • nclayout: Vollständige NC-Tastatur

x, y

Position der linken oberen Ecke der virtuellen Tastatur, bezogen auf die linke obere Ecke des Frames, in dem die Tastatur angezeigt wird.

KeyHeight

Höhe (und Breite) der Tasten der virtuellen Tastatur.

Wenn in einer Display-Konfiguration weitere Frames mit anderen Größen konfiguriert werden, in denen die virtuelle Tastatur angezeigt werden soll, muss die Datei sldm_keyboard.ini um entsprechende Abschnitte erweitert werden und im oem-Verzeichnis abgelegt werden.

Virtuelle MCP

Im Display-Manager kann die MCP in zwei verschiedenen Größen verwendet werden:

  • 760x505

  • 1024x242 - Wird im weiteren mit der Kennung "below" gekennzeichnet

Wenn der Display-Manager vier Fenster hat und die Bedienoberfläche rechts oben dargestellt wird, dann wird die MCP zum einen in der Größe [760x505] links unten und zum anderen in der Größe [1024x242] unterhalb der Bedienoberfläche dargestellt.

Weitere Informationen finden Sie parallel im Kapitel "Sidescreen Maschinensteuertafel anzeigen".

Vorgehensweise

  1. Kopieren Sie die Musterdateien sldm_1920x1080.ini, sldmmcppage1.ini, sldmmcppagebelow1.ini, systemconfiguration.ini, slsidescreenmcppageconfig.xml, slsidescreenmcppage_deu.ts und die Unterverzeichnisse ico640 und ico1920 aus dem Verzeichnis:

    /siemens/sinumerik/hmi/template/displaymanager/mcp/

  2. Legen Sie die Kopien von sldm_1920x1080.ini, sldmmcppage1.ini, sldmmcppagebelow1.ini und systemconfiguration.ini​·​in das Verzeichnis:

    /oem/sinumerik/hmi/cfg oder

    /user/sinumerik/hmi/cfg

  3. Legen Sie die Kopie von slsidescreenmcppageconfig.xml in das Verzeichnis:

    /oem/sinumerik/hmi/proj oder

    /user/sinumerik/hmi/proj

  4. Legen Sie die Kopie von slsidescreenmcppage_deu.ts in das Verzeichnis:

    /oem/sinumerik/hmi/lng oder

    /user/sinumerik/hmi/lng

    Hinweis

    Die TS-Datei wird im Hochlauf automatisch vom System integriert.

  5. Legen Sie die Verzeichnisse ico640 und ico1920 unter folgendem Pfad ab:

    /oem/sinumerik/hmi/ico oder

    /user/sinumerik/hmi/ico

  6. Die Maschinensteuertafeln können nach einem Hochlauf angewählt werden.

Optional können weitere Maschinensteuertafel-Fenster aktiviert werden (in diesem Beispiel vier weitere):

  1. Nehmen Sie folgende Anpassungen in der Datei systemconfiguration.ini vor:

    Entfernen Sie das ";" vor den Schlüsselwörtern DLG501, DLG502, DLG503 und DLG504.

  2. Nehmen Sie folgende Anpassungen in der Datei sldm_1920x1080.ini vor:

    Entfernen Sie das ";" vor den Schlüsselwörtern MENUITEM500, MENUITEM501, MENUITEM502, MENUITEM503, MENUITEM550, MENU550 und FRAME500.

    Bei folgenden Schlüsselwörtern fügen Sie ein ";" vor dem ersten Auftreten ein und entfernen das ";" eine Zeile tiefer:

    MENUITEM124, MENUITEM125, MENU022, MENU032, MENU033, FRAME022 und FRAME028.

  3. Kopieren Sie die Musterdateien sldmmcppage2.ini, sldmmcppagebelow2.ini, sldmmcppagebelow3.ini und sldmmcppagebelow4.ini in das Verzeichnis:

    /oem/sinumerik/hmi/cfg oder

    /user/sinumerik/hmi/cfg

Applikation zur Anzeige von Widgets

Für die Projektierung im Standard wird in der Datei systemconfiguration.ini folgendes deklariert:

[dialogs]DLG109= name:=SlWidgetsApp, implementation:=sldmsidescreenapp.SlSideScreenDialog, process:=SlHmiHost1, preload:=false, cmdline:="-sidescreen1 sldmwidgets1.ini -sidescreen2 sldmwidgets2.ini -spacing 3"

Für die Projektierung innerhalb einer Display-Konfiguration zu verwendender Applikationsname:

SlWidgetsApp

Die Applikation darf innerhalb einer Display-Konfiguration nur in einer Instanz verwendet werden.

Kameraanzeige

In den Dateien sldmwidgets1.ini und sldmwidgets2.ini wird folgendes Schlüsselwort für die Applikationsprojektierung verwendet:

ELEMENT005

Wenn über SINUMERIK Operate Kameras eigerichtet sind, können bis zu zwei Übertragungen im Display Manager angezeigt werden.

WEB Browser

In der Datei sldm_1920x1080.ini wird folgendes Schlüsselwort für die Applikationsprojektierung verwendet:

MENUITEM129

Weitere Informationen, wie Sie den WEB Browser einrichten, finden Sie im Kapitel "WEB Browser-Konfigurationsparameter".

SINUEMRIK Operate

Für die Projektierung innerhalb einer Display-Konfiguration zu verwendender Applikationsname:

OPERATE

Standarddatei sldm_1920x1080.ini:

FRAME040= name:=fMax, x:=66, y:=66, width:=1854, height:=1014, app:=OPERATE

SINUMERIK Operate darf innerhalb einer Display-Konfiguration nur in einer Instanz verwendet werden.

Außerdem ist zu beachten, dass SINUMERIK Operate nur in bestimmten Größen angezeigt/ausgeführt werden kann. Die aktuell unterstützten Größen (in Pixel) sind:

  • 640x480

  • 760x505

  • 800x480

  • 800x600

  • 1024x768

  • 1024x1014

  • 1080x810

  • 1280x768

  • 1280x800

  • 1280x1024

  • 1300x768

  • 1366x768

  • 1440x1080

  • 1454x1080

  • 1854x1014

  • 1854x1080

  • 1920x1080

Beispiel für Applikationen vom Typ B

Projektierung in der Datei systemconfiguration.ini am Beispiel der Applikation notepad.exe:

[processes]PROC500=process:=Notepad, image:=notepad, cmdline:="C:\Windows\system32\notepad.exe", oemframe:=true, windowname:="Unbenannt - Editor", classname:="Notepad", processaffinitymask:=0xFFFFFFFF, deferred:=true

Für die Projektierung innerhalb einer Display-Konfiguration zu verwendender Applikationsname:

Notepad

Die Applikation darf innerhalb einer Display-Konfiguration nur in einer Instanz verwendet werden.

Projektierung in der Datei oemframe.ini am Beispiel der Applikation notepad.exe:

[notepad]nPlacementMode=3WindowStyle_Off = 15597568

Applikationen vom Typ B dürfen jeweils nur in einer Instanz ausgeführt werden.

Mögliche Fehlfunktion

Eingebundene OEM-Applikationen können inkompatibel sein und Fehlfunktionen verursachen. Prüfen Sie die Projektierung bei Fehlfunktionen.
