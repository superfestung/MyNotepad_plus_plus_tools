Zentrale Hersteller-Domäne verbinden

Einleitung

Wenn Sie additiv zu ihrer lokalen Hersteller-Domäne diese Domäne mit einer zentralen Benutzerverwaltung verbinden möchten, können Sie die Einstellungen über den Dialog "Hersteller-Domäne mit einer zentralen Benutzerverwaltung verbinden" parametrieren.

Die Verbindung der Domäne mit einer zentralen Benutzerverwaltung hat den Vorteil, dass die Benutzer zentral auf einem UMC-Server verwaltet werden können. Wenn sich ein Benutzer der zentralen Benutzerverwaltung anmeldet, erhält er die Rechte entsprechend der lokalen Zuordnung seiner zentralen Benutzergruppen zu Maschinenrollen.

Voraussetzung

  • Für das Einrichten einer zentrale Benutzerverwaltung (UMC) benötigen Sie eine der verfügbaren elektronischen Lizenzen:

    • 6ES7823-1UE10-0YA0 (Lizenzzertfiikat für 4000 Benuzterkonten)

    • 6ES7823-1UE30-0YA0 (Lizenzzertifikat für 100 Benutzerkonten)

  • Die UMC-Server Version muss 2.13.1 oder höher sein.

  • Die Benutzerverwaltung ist aktiviert.

  • Die lokale Domäne für den Hersteller ist eingerichtet und der Security Admin für den Hersteller ist eingerichtet.

  • Sie verfügen über die Rolle "Security Admin Hersteller".

Hinweis

Beachten Sie, dass alle bereits angelegten Benutzer in der lokalen Domäne aktiv bleiben und weiterhin für Anmeldungen verwendet werden können.

  • Stellen Sie sicher, dass diese Benutzer-Konten gepflegt werden, oder löschen Sie diese, wenn diese nicht mehr benötigt werden, um Missbrauch zu vermeiden.
Hinweis

Beachten Sie, dass alle lokalen Benutzergruppen gelöscht werden, die keine namentliche Entsprechung auf der Domäne der zentralen Benutzerverwaltung besitzen, sobald Sie ihre aktive Domäne mit einer zentralen Benutzerverwaltung verbinden.

D.h., bevor Sie die zentrale Domäne verbinden, stellen Sie folgendes sicher:

  • Dass Sie die Admin Rechte entweder direkt über eine Zuweisung zu der entsprechenden Maschinenrolle beziehen
    ODER

  • dass die Benutzergruppe, über die Sie Admin-Rechte haben, in der zentralen Domäne namensgleich existiert.

Andernfalls wird die Verbindung zur zentralen Domäne nicht erfolgreich sein.

Vorgehensweise

Hersteller-Domäne mit zentraler Benutzerverwaltung verbinden

  1. Im Bedienbereich "Inbetriebnahme" rufen Sie die Benutzerverwaltung über die Softkeys "Security > Benutzerverwaltung" auf. Die Übersichtsseite zur Benutzerverwaltung öffnet sich im Softkey "Konfiguration".

  2. Klicken Sie auf die Schaltfläche "Verbinden" neben der lokalen Hersteller-Domäne.

  3. Vergeben Sie einen Namen für die zentrale Hersteller-Domäne.

  4. Geben Sie die UMC-Server-Adresse des zentralen UMC-Servers ein.

  5. Bestätigen Sie ihre Eingaben mit "OK".

  6. Prüfen Sie den Fingerprint des UMC-Servers.

  7. Um die Verbindung zu bestätigen, müssen Sie sich mit den Login-Daten des zentralen Administrators autorisieren. Geben Sie Benutzername und Passwort des zentralen Administrators ein und bestätigen Sie den Login-Dialog mit "Verbinden".

Ergebnis

Sie haben nun die lokale Hersteller-Domäne mit einer zentralen Benutzerverwaltung verbunden. Zentrale Benutzer des UMC-Servers können nun über zentrale Benutzergruppen den entsprechenden Maschinenrollen zugeordnet werden und sich am System mit den entsprechenden Rechten anmelden.

Sie sind weiterhin als lokaler Admin Hersteller im System eingeloggt.

Weitere Schritte

  • Sie können vorhandene zentrale Benutzergruppen nutzen und diese Maschinenrollen zuordnen.

  • Sie können zentrale Benutzergruppen anlegen und diese Maschinenrollen zuordnen.

  • Sie können lokale Benutzer anlegen und diese zentralen Benutzergruppen zuordnen.

  • Sie können die Hersteller-Domäne von der zentralen Benutzerverwaltung trennen.
