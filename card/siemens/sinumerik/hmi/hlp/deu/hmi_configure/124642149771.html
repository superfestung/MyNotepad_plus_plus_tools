Trace-Parameter

Über das Eingabefeld "Parameter (tcpdump)" im Fenster Netzwerk-Trace haben Sie die Möglichkeit, die Bedingungen der Protokollaufzeichnung selbst zu parametrieren.

Syntax

<Schalter> <Option>

Parameter <Schalter>

Wert

Bedeutung

-D

Zeigt eine Liste aller Netzwerk-Schnittstellen. Alle anderen <Schalter>-Parameter werden ignoriert.

-i <Schnittstelle>

Definiert, welche Netzwerk-Schnittstelle geprüft werden muss.

-n

Hostnamen werden nicht aufgelöst.

-c <Anzahl>

Die Aufzeichnung wird nach einer definierten Paketanzahl beendet.

-C <Dateigröße>

Begrenzt die Dateigröße. Ist die Dateigröße erreicht, wird eine neue Datei mit Namen und fortlaufender Ziffer, beginnend mit 1 angelegt. Die Einheit der Dateigröße entspricht Millionen Bytes.

-e

Gibt den Ethernet-Header aus.

-q

Eine Protokoll-Kurzfassung wird erstellt.

-vv

Es werden detailliertere Paketinformationen erstellt.

-S

Umstellung der Paketsequenznummer von absolut auf relativ.

-A

Gibt den Inhalt eines Pakets als ASCII aus.

-xx

Gibt den Inhalt eines Pakets als Hex aus.

-XX

Gibt den Inhalt eines Pakets sowohl als Hex wie auch als ASCII aus.

-s <Bytes>

Gibt die Paketgröße in Bytes an, die pro Paket aufgezeichnet werden muss. Standard sind 68 Bytes.

Hinweis

Im Aufruf ist immer implizit der Wert -w zur Erstellung der Standardprotokolldatei enthalten:

HMI-Daten/Protokolle/Netzwerktrace/tcpdump.pcap bzw. windump.pcap

Parameter <Option>

Option

Wert

Bedeutung

host

IP-Adresse oder Hostname

Es werden nur Pakete aufgezeichnet, die entweder als Quelle oder als Ziel die angegebene IP-Adresse oder den Hostnamen besitzen.

net

Netzwerkbereich

Es werden nur Pakete aufgezeichnet, die entweder als Quelle oder als Ziel eine IP-Adresse oder den Hostnamen aus dem angegebenen Netzwerkbereich besitzen.

port

eine Zahl von 0 bis 65535

Zeichnet Pakete auf, die entweder auf Quell- oder auf Zielseite den angegebenen Port besitzen, z. B. 5900.

portrange

einen Bereich zwischen 0 bis 65535

Zeichnet Pakete auf, die entweder auf Quell- oder auf Zielseite einen Port in dem angegebenen Portbereich besitzen.

src

host, net, port oder portrange

Es werden nur Pakete aufgezeichnet, die als Quelle den angegebenen Wert besitzen.

dst

host, net, port oder portrange

Es werden nur Pakete aufgezeichnet, die als Ziel den angegebenen Wert besitzen.

proto

ether, fddi, tr, wlan, ip, ip6, arp, rarp, decnet, tcp und udp

Netzwerkprotokoll, in dem Pakete aufgezeichnet werden können.

Sie können mehrere Optionen durch folgende Ausdrücke miteinander verknüpfen:

and, or, not

Erweiterte Ausdrücke können Sie durch folgende Zeichen abgrenzen:

( ) und '

Referenz

Weitere Parametrierungen finden Sie unter www.tcpdump.org.

Beispiele

-n port 22

; Zeichnet auf, ob Pakete auf Port 22 des lokalen Systems ankommen.

-i eth0 port 4080

; Zeichnet alle an der Schnittstelle eth0 ankommenden Daten für Port 4080 auf.

-i 2

; Zeichnet alle an der Schnittstelle mit der Adapternummer 2 ankommenden Daten auf.

-n host 10.113.20.0

; Zeichnet den Netzwerkverkehr von System 10.113.20.0 auf, wobei die IP-Adresse nicht in den Hostnamen umgewandelt wird.

192.168.214.4 and not net 10.113.20.0

; Zeichnet den gesamten Netzwerkverkehr des Systems 192.168.214.4 auf, der nicht über das System 10.113.20.0 verläuft.

-i 10.113.20.0 -q '(tcp port 80) or (tcp port 443)'

; Zeichnet sowohl HTTP-, als auch HTTPS-Pakete der Netzwerkschnitstelle vom System 10.113.20.0 mit.
