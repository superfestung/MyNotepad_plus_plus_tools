Maschinenrollen anlegen

Einleitung

Wenn Sie eine neue Maschinenrolle anlegen möchten, nutzen Sie den Dialog "Maschinenrollen anlegen".

Voraussetzung

  • Sie verfügen über die Rolle "Security Admin Hersteller".

Vorgehensweise

  1. Im Bedienbereich "Inbetriebnahme" rufen Sie die Benutzerverwaltung über die Softkeys "Security > Benutzerverwaltung" auf.

    Hinweis

    Alternativ: Standard-Maschinenrollen nutzen

    Wenn Sie keine komplett neue Maschinenrolle anlegen möchten, sind standardmäßig bereits für jeden AccessLevel und Security Admin Maschinenrollen vorangelegt. Mit der Rolle "Security Admin Hersteller" können Sie alternativ auch diese Rollen übernehmen, ändern oder löschen.

  2. Klicken Sie auf den Softkey "Maschinenrollen". Klicken Sie auf der Übersichtsseite auf "+ Maschinenrolle", um eine neue Maschinenrolle anzulegen. Ein Pop-Up Dialog öffnet sich.


    Neue Maschinenrolle anlegen

  3. Vergeben Sie einen beliebigen Namen für die Maschinenrolle und eine möglichst treffende Beschreibung, welche die Rechte der Maschinenrolle beschreibt.

    Hinweis

    Beachten Sie, dass bei der Vergabe des Namens für die Maschinenrolle Sonderzeichen nicht erlaubt sind.

  4. Wählen Sie über das Optionskästchen "darf vom Endkunden verwendet werden", ob der Anwender diese Maschinenrolle auch verwenden/vergeben darf. Dies gilt für alle Maschinenrolle mit Ausnahme der Maschinenrolle "Security Admin". Diese Rolle kann nicht geteilt werden.

    Hinweis

    Bedenken Sie als Hersteller, dass der Security Admin Anwender nur diejenigen Rollen vergeben kann, bei denen hier der Haken gesetzt ist.

    • Planen Sie deswegen vorher sorgfältig, welche Rollen/Rechte nur dem Hersteller vorbehalten sein sollten.

  5. Wähen Sie eine SINUMERIK-Rolle aus der Liste aus.

  6. Wenn Sie bereits Benutzergruppen angelegt haben, können Sie festlegen, welche Benutzergruppe(n) der Maschinenrolle zugeordnet werden soll(en).

    • Wenn einer Benutzergruppe mehrere Maschinenrollen mit unterschiedlichen SINUMERIK-Zugriffstufen zugeordnet sind, dann erhält der Benutzer immer die Rechte der Maschinenrolle mit der höchsten Zugriffstufe.

    • Die Vergabe von mehreren Maschinenrollen an eine Benutzergruppe ist daher nur sinnvoll, wenn diese die SINUMERIK-Zugriffstufe ergänzen, z.B. durch die SINUMERIK Rolle "Security Admin" oder indem die Maschinenrollen andere Rechte in anderen Teilen des Systems haben (z.B. OA Applikationen, PLC, etc.).

  7. Klicken Sie auf "OK", um die neue Maschinenrolle in die Liste der Maschinenrollen zu übernehmen. Legen Sie bei Bedarf weitere Maschinenrollen an, indem Sie die Schritte 2-5 wiederholen.

Ergebnis

Eine oder mehrere Maschinenrollen sind nun angelegt und werden in der Liste der Maschinenrollen angezeigt.
