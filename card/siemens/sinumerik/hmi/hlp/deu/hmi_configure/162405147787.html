NFS-Client konfigurieren

Um die Steuerung und das PC-System im Netzwerk mit einem externen NFS-Server zu verbinden, wird das NFS-Serverprotokoll (Network File System) verwendet. Verschiedene Server erfordern je nach Konfiguration eine andere Protokollversion, um eine Verbindung mit der NCU und dem PC-System zu ermöglichen.

Kompatibilität

Die NFS-Protokollversionen sind kompatibel für folgende Software:

CNC-Software

Unterstützte NFS-Protokollversion

V4.5 (828D, 840D sl)

V4.7 (828D, 840D sl)

V4.8 (828D, 840D sl)

V2.0, V3.0

V4.9 (828D, 840D sl)

V2.0, V3.0, V4.0, V4.1, V4.2

V6.1x (ONE)

V6.2x (ONE)

NFS-Serververbindung in SINUMERIK Operate projektieren

Für die Projektierung des NFS-Servers als Softkey im Programm-Manager steht Ihnen im Bedienbereich
"Inbetriebnahme > HMI > Log. Laufw." das Fenster "Laufwerke einrichten" zur Verfügung.

Richten Sie den NFS-Server als Linux-Netzlaufwerk "NW Linux" ein.

 

Weitere Informationen zur Laufwerksprojektierung finden Sie in der Online-Hilfe von SINUMERIK Operate.

NFS-Serverkonfiguration überprüfen

  1. Um die NFS-Serverkonfiguration zu prüfen, rufen Sie PuTTY auf.

  2. Melden Sie sich als Benutzer "manufact" an.

  3. Führen Sie den Befehl mount | grep nfs aus.

    Sie erhalten folgende Ausgabe, die ähnliche Angaben enthält wie z. B.:

    192.168.214.241/carlshare auf /tmp/.logdrived_mnt. DEV_6 typ cifs (rw,nosuid,nodev,relatime,vers=3,...)

    Das Beispiel zeigt, dass die NFS-Freigabe mit dem NFS-Protokoll V3 für den Server aktiviert wurde.

Hinweis

NFS handelt immer die bestmögliche Protokollversion zwischen Client und Server aus.
