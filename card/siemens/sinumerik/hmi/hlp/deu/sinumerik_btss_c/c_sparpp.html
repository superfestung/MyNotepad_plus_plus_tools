Baustein C/SPARPP: Programmzeiger im Automatikbetrieb

Im Automatikbetrieb kann aus der Hauptprogrammebene in mehrere Unterprogrammebenen verzweigt werden. Der Zustand der Programmbearbeitung kann für jede Programmebene ermittelt werden. Jede Variable des Bausteins besteht aus 18 Zeilen. Damit können die Hauptprogrammebene und 17 Unterprogrammebenen (inkl. ASUP-Ebenen) adressiert werden.

Die Feldindizes (Zeilenindizes) haben folgende Bedeutung:

1 = Hauptprogrammebene
2-18 = Unterprogrammebenen

eesBufferFilling Anzahl Bytes für Interpeter
Spaltenindex:
25 		Zeilenindex:
Prog.ebene
Index der Programmebene 		Max. Zeilenindex:
18
Der Wert ist nur bei EES relevant, bei dem das Teileprogramm partiell in einen Buffer in den NCK geladen wird.
Er gibt an, wieviele Bytes dem Interpreter zur Bearbeitung im Buffer zur Verfügung stehen (eesBufferEnd - byteOffsetVL).
Wenn der Wert gegen 0 geht, ist das ein Indiz, das der Nachladevorgang nicht schnell genug ist, um den Interpreter schnell genug mit NC-Sätzen zu versorgen.
Typ
TYPE_DWORD 		Einheit
keine Einheit 		Standardwert
 Minimalwert
 Maximalwert
Zugriffspfad HMI: /Channel/ProgramPointer/eesBufferFilling[u<Kanalnummer>, <Zeilenindex>]

lastBlockNoStr Zuletzt programmierte Satznummer
Spaltenindex:
13 		Zeilenindex:
Index der Programmebene 		Max. Zeilenindex:
18
Liefert die zuletzt programmierte Satznummer für jede
Programmebene, wenn $MN_DISPLAY_FUNCTION_MASK Bit 0
gesetzt ist. Eine Satznummer wird solange angezeigt, bis
entweder eine neue Satznummer programmiert wird oder die
Unterprogrammebene, die die Satznummer generiert hat, verlassen
wurde. Satznummern ausgeblendeter Sätze werden nicht angezeigt.
Ist DISPLOF aktiv, erfolgt ebenfalls keine Anzeige.
Typ
TYPE_STRING 		Einheit
keine Einheit 		Standardwert
 Minimalwert
 Maximalwert
Zugriffspfad HMI: /Channel/ProgramPointer/lastBlockNoStr[u<Kanalnummer>, <Zeilenindex>]
