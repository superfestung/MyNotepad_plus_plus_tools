SafeUserData - Import

Aus einer gewählten Importdatei können SUD-Aktualwerte in die PLC importiert werden.

Voraussetzungen

  • Der in der Datei enthaltene Hashwert ist korrekt und zeigt die Unversehrtheit der Datei an.

  • Typ und Reihenfolge der Elemente in den Anwenderbausteinen stimmen mit dem geladenen PLC-Projekt überein.

    Kleinere Unterschiede beim Elementnamen werden toleriert.

  • Alle "writeEnable"-Signale sind aktiv.

Aufruf

MENU SELECT > Inbetriebnahme > Menüfortschalttaste > Safety > SafeUserData > Import

Anschließend öffnet sich ein Dialog zur Auswahl der SUD-Importdatei.

Informationen

Name

Beschreibung

SafeUserData importieren

Wählen Sie hier die gewünschte Importdatei aus.

Vertikale Softkeys

Name

Beschreibung

OK

Mit diesem Softkey starten Sie den Import der SUD-Aktualwerte aus der gewählten Importdatei. Das Programm prüft die Daten der Importdatei und importiert alle Bausteine mit identischen Symbonamen.

Bei Inkonsistenzen zwischen den Symbolnamen der vorhandenen und der zu importierenden Daten erscheint eine Abfrage. Bestimmen Sie, wie mit diesen Inkonsistenzen verfahren werden soll.

Im Anschluss daran wird das Importergebnis angezeigt.

Mit dem gleichen Softkey bestätigen Sie das angezeigte Importergebnis.

Abbruch

Mit diesem Softkey brechen Sie den Import der Aktualwerte ab.

Nach dem Abbruch wird die Maske "SafeUserData - Übersicht" angezeigt.

Ergebnis

Das Programm schreibt alle Instanzen und überprüft dabei die Aktualwerte.

Hinweis

Abnahmetest erforderlich

Führen Sie nach dem Import von Aktualwerten immer einen Abnahmetest durch.

  • War die Überprüfung erfolgreich, erscheint noch eine Maske mit der Zusammenfassung der Überprüfung. Der Import kann mit "OK" gestartet oder mit "Abbruch" abgebrochen werden. Nach Ende des Imports erscheint die Meldung "Der Import der SafeUserData wird beendet".

  • War der Import dagegen nicht erfolgreich, erhalten Sie die Meldung "Der Import der SafeUserData wurde abgebrochen". Mögliche Ursachen sind:

    • Beim Lesen der Importdatei ist ein Fehler aufgetreten.

    • Der Hashwert der Datei stimmt nicht.

    • Die Version der Datei ist ungültig.

    • Anzahl/Typ/Name der Anwenderbausteine oder Reihenfolge der Elemente eines Bausteins stimmen nicht mit dem aktuellen PLC-Projekt überein.

    • Das "writeEnable"-Signal ist nicht bei allen Anwenderbausteinen aktiv.

  • Hat die Überprüfung ergeben, dass die Importdatei weniger Bausteine enthält als der aktive SafeUserData-Datensatz, so werden alle identischen Bausteine aus der Importdatei importiert. Bausteine, die nur im aktiven SafeUserData-Datensatz enthalten sind, werden nicht überschrieben.

  • Hat die Überprüfung ergeben, dass die Importdatei mehr Bausteine enthält als der aktive SafeUserData-Datensatz, so werden alle identischen Bausteine aus der Importdatei importiert. Bausteine, die nur in der Importdatei enthalten sind, werden nicht importiert.

  • Hat die Überprüfung ergeben, dass sich die Symbolnamen der aktiven SafeUserData-Datensätze von den Symbolnamen der Importdatei unterscheiden, wird dies in einer Meldung angezeigt (siehe SafeUserData - Übersicht - Details). Das weitere Vorgehen können Sie über die Softkeys definieren:

    • Softkey Abbruch

      Sie können den Import der Importdatei komplett abbrechen.

    • Softkey Verwerfen

      Sie bewerten der Unterschiede (Softkey OK), die in einer Maske angezeigt werden. Sie entscheiden Sie sich dafür, die Werte der abweichenden Symbolnamen für die angezeigte Instanz nicht zu übernehmen. In diesem Fall benutzen Sie den Softkey Verwerfen. Liegen weitere Unterschiede für andere Instanzen vor, werden diese anschließend in Folgemasken angezeigt.

    • Softkey Übernehmen

      Sie bewerten der Unterschiede (Softkey OK), die in einer Maske angezeigt werden. Sie entscheiden Sie sich dafür, alle Werte der abweichenden Symbolnamen aller Instanzen aus der Importdatei zu übernehmen.

    Eine Übersicht zeigt anschließend nochmals, welche Entscheidung Sie getroffen haben. Mit dem Softkey OK können Sie den entgültigen Import anstoßen.
